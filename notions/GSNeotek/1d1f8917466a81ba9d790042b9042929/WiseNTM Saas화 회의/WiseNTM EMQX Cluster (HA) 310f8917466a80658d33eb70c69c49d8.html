WiseNTM EMQX Cluster (HA)

구축 가이드 및 작업 내역

1. 인프라 환경 개요 (Environment)

  • OS: Amazon Linux 2023 (ARM64)
  • Instance Type: t4g.medium (추후 운영 시 m7g.large 이상 활용할 계획임)
  • Architecture: 2 Nodes Active-Active Cluster (AZ-a, AZ-c 이중화 분산)
  • Node 정보:
    • Broker 1 (AZ-a): Private IP 172.31.43.118
    • Broker 2 (AZ-c): Private IP 172.31.44.207

2. 네트워크 및 보안 그룹 (Security Group) 설정

클러스터 노드 간 데이터 동기화와 단말기 연결을 위해 전용 보안 그룹(EMQX-Cluster-SG)을 생성하고 두 인스턴스에 공통 적용했습니다.

[인바운드 규칙 상세]

포트(Port)프로토콜소스(Source)목적 / 설명 (Description)
22TCP관리자 IPSSH access for Admin
18083TCP관리자 IPEMQX Web Dashboard
1883TCP0.0.0.0/0MQTT TCP Port for Devices (NLB 연동 포트)
8083TCP0.0.0.0/0MQTT WebSocket Port
4370TCPsg-xxxxxxxx (자기 참조)Erlang distributed node communication (클러스터 동기화)
5369TCPsg-xxxxxxxx (자기 참조)Erlang epmd for cluster discovery (노드 탐색)
포인트: 4370, 5369 포트의 소스를 IP가 아닌 자신의 보안 그룹 ID(자기 참조)로 설정하여, 해당 보안 그룹이 부여된 서버들끼리만 완벽하게 격리된 상태로 통신하도록 보안을 강화했습니다.

3. EMQX 설치

Amazon Linux 2023(ARM64) 환경에 맞춰 RPM 패키지 방식으로 설치를 진행했습니다. (양쪽 서버 동일 적용)

# 1. EMQX Repository 추가
curl -s https://assets.emqx.com/scripts/install-emqx-rpm.sh | sudo bash

# 2. EMQX 설치 (dnf)
sudo dnf install emqx -y

# 3. 서버 재부팅 시 자동 시작 등록
sudo systemctl enable emqx

4. 정적 클러스터링 설정

EMQX 5.x HOCON 포맷을 사용하여 고정 IP 기반의 정적(Static) 클러스터링을 구성했습니다.

설정 파일 경로: /etc/emqx/emqx.conf

[Broker 1 (172.31.43.118) 설정]

node {
  name = "emqx@172.31.43.118"
  cookie = "wisentm_secret_cookie"
  data_dir = "/var/lib/emqx"
}

cluster {
  name = emqxcl
  discovery_strategy = static
  static {
    seeds = ["emqx@172.31.43.118", "emqx@172.31.44.207"]
  }
}

dashboard {
    listeners {
        http.bind = 18083
    }
}

[Broker 2 (172.31.44.207) 설정]

node {
  name = "emqx@172.31.44.207" # 자신의 IP로 변경
  cookie = "wisentm_secret_cookie" # Broker 1과 동일한 쿠키 사용
  data_dir = "/var/lib/emqx"
}

cluster {
  name = emqxcl
  discovery_strategy = static
  static {
    seeds = ["emqx@172.31.43.118", "emqx@172.31.44.207"]
  }
}

dashboard {
    listeners {
        http.bind = 18083
    }
}

5. 서비스 실행 및 검증

1) 서비스 재시작

두 서버에서 설정을 마친 후 서비스를 재시작하여 클러스터를 결합했습니다.

sudo systemctl restart emqx

2) 클러스터 상태 확인 (CLI)

sudo emqx ctl cluster status

  • 성공 결과: running_nodes 목록에 두 개의 IP(emqx@172.31.43.118, emqx@172.31.44.207)가 모두 노출됨을 확인했습니다.

3) 웹 대시보드(GUI) 확인

  • http://[퍼블릭IP]:18083 접속
  • 좌측 [Cluster] -> [Nodes] 메뉴에서 2개의 노드가 Running 상태로 정상 연동되어 리소스를 공유하는 것을 최종 확인
  • 아이디 : admin
  • 비밀번호 : [비밀번호-마스킹]
