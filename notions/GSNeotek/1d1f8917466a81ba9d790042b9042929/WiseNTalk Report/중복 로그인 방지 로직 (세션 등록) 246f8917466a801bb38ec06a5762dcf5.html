중복 로그인 방지 로직 (세션 등록)

소프트폰 중복 로그인 방지 시스템

개요

소프트폰 앱에서 서로 다른 브라우저에서 동시 로그인을 방지하는 시스템입니다.

  • 한 명의 사용자는 단 하나의 브라우저 탭에서만 소프트폰 사용 가능
  • Chrome, Edge, Firefox 등 다른 브라우저 간에도 중복 로그인 차단

동작 원리

1. 세션 등록 과정

sequenceDiagram
    participant C as Chrome 탭
    participant S as Session Server
    participant E as Edge 탭

    C->>S: 세션 등록 요청 (userId: 12345)
    S->>S: 기존 세션 확인
    Note over S: 기존 세션 없음
    S->>C: 응답: isLeader = true ✅
    C->>C: 리더가 됨 → 소프트폰 정상 동작

    E->>S: 세션 등록 요청 (userId: 12345)
    S->>S: 기존 세션 확인
    Note over S: Chrome 세션 이미 존재
    S->>E: 응답: isLeader = false ❌
    E->>E: 팔로워가 됨 → 중복 로그인 메시지 표시

2. 하트비트 시스템

sequenceDiagram
    participant C as Chrome (리더)
    participant S as Session Server
    participant E as Edge (팔로워)

    loop 3초마다
        C->>S: 하트비트 전송 
        S->>C: OK
    end

    loop 5초마다
        E->>S: 리더 상태 확인 
        S->>E: 리더 여전히 활성화됨
    end

3. 리더 교체 과정

sequenceDiagram
    participant C as Chrome (리더)
    participant S as Session Server
    participant E as Edge (팔로워)

    Note over C: 브라우저 종료
    C->>S: 세션 정리 요청
    S->>S: Chrome 세션 삭제

    E->>S: 리더 상태 확인
    S->>E: 리더 없음, 승계 가능
    E->>S: 리더십 요청
    S->>E: 새로운 리더 승인 
    E->>E: 페이지 새로고침 → 정상 동작

클라이언트 (리액트 앱)

ServerSessionManager 클래스

class ServerSessionManager {
  constructor(apiUrl, userId, userEmail)

  // 주요 메서드
  async registerSession()      // 세션 등록
  startHeartbeat()            // 하트비트 시작 (리더만)

}

설치 및 실행

1️⃣ API 서버 설정

# 1. 새 디렉토리 생성
mkdir session-api-server
cd session-api-server

# 2. 프로젝트 초기화
npm init -y

# 3. 패키지 설치
npm install express cors dotenv
npm install --save-dev nodemon

# 4. server.js 파일 생성 (위의 코드 사용)

# 5. package.json scripts 수정
{
  "scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js"
  }
}

# 6. 서버 실행
npm run dev

2️⃣ 리액트 앱 설정

# 1. .env 파일에 API URL 추가
echo "REACT_APP_SESSION_API_URL=http://localhost:3001/api" >> .env

# 2. serverSessionManager.js 파일 추가

# 3. Zendesk.js 수정

# 4. 리액트 앱 실행
npm start

테스트 방법

시나리오 1: 정상 동작 테스트

  1. Chrome으로 localhost:3000 접속
    • ✅ 소프트폰 정상 동작 (리더)
    • 콘솔: "Session XXX is LEADER"
  1. Edge로 localhost:3000 접속
    • ❌ "중복 로그인" 메시지 표시 (팔로워)
    • 콘솔: "Session XXX is FOLLOWER"

시나리오 2: 리더 교체 테스트

  1. Chrome 탭 종료
  1. Edge에서 자동으로 리더 승계
  1. Edge 페이지 새로고침 후 정상 동작

디버깅 도구

  • 세션 목록 확인: http://localhost:3001/api/session/list
  • 서버 상태 확인: http://localhost:3001/health
  • 브라우저 콘솔: 상세한 로그 메시지

주요 특징

장점

  • 완전한 중복 방지: 서로 다른 브라우저 간에도 동작
  • 자동 리더 교체: 리더가 종료되면 자동으로 새 리더 선출
  • 실시간 감지: 하트비트로 실시간 상태 모니터링
  • 복구 가능: 네트워크 문제 후 자동 복구

타이밍

  • 하트비트 주기: 3초 (리더 → 서버)
  • 상태 체크 주기: 5초 (팔로워 → 서버)
  • 세션 타임아웃: 10초 (응답 없으면 만료)
  • 세션 정리: 5초마다 만료된 세션 자동 삭제

안전장치

  • 에러 처리: 네트워크 오류 시 안전하게 팔로워로 전환
  • 메모리 관리: 만료된 세션 자동 정리
  • 브라우저 호환: 모든 모던 브라우저 지원
  • 세션 복구: 탭 활성화 시 상태 재확인

문제 해결

일반적인 문제

1. "중복 로그인"이 계속 표시될 때

  • 서버가 실행 중인지 확인: http://localhost:3001/health
  • 활성 세션 확인: http://localhost:3001/api/session/list
  • 브라우저 콘솔에서 에러 메시지 확인

2. 리더 교체가 안 될 때

  • 이전 리더 탭이 완전히 종료되었는지 확인
  • 5-10초 정도 기다린 후 팔로워 탭 새로고침

3. API 연결 오류

  • CORS 설정 확인
  • 방화벽/보안 프로그램 확인
  • 네트워크 연결 상태 확인

로그 메시지 이해

메시지의미

❌ FOLLOWER session팔로워 세션 (중복 로그인)

확장 가능성

프로덕션 환경 개선사항

  1. Redis 사용: 메모리 기반 → Redis로 세션 저장
  1. 로드밸런싱: 여러 API 서버 인스턴스
  1. 보안 강화: JWT 토큰, HTTPS 적용
  1. 모니터링: 세션 로그, 알림 시스템
  1. DB 저장: 세션 히스토리 관리

고급 기능

  • 강제 로그아웃: 관리자가 특정 세션 강제 종료
  • 세션 제한: 사용자별 최대 세션 수 제한
  • 지역별 분산: 다중 데이터센터 지원
  • 실시간 알림: WebSocket으로 즉시 상태 변경 알림
