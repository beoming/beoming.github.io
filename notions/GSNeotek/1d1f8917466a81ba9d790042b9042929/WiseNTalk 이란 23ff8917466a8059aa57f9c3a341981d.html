WiseNTalk 이란?

이 애플리케이션은 React를 기반으로 구축된 웹 소프트폰(Soft Phone) 솔루션으로 보입니다. 주요 기능은 상담원 상태 관리, 음성/채팅/태스크를 통한 고객 상담, Zendesk 연동, 그리고 Amazon Connect와의 연동을 통한 통신 기능입니다.

전체적인 코드베이스는 모듈화가 잘 되어 있으며, Redux를 사용하여 애플리케이션의 상태를 효율적으로 관리하고 있습니다.

애플리케이션 구조 및 로직 분석

이 애플리케이션은 다음과 같은 주요 모듈과 계층으로 구성됩니다:

  1. 진입점 (Entry Point)
    • src/index.js: React 애플리케이션의 메인 진입점으로, ReactDOM.render를 사용하여 App 컴포넌트를 렌더링합니다.
    • src/App.js: 애플리케이션의 최상위 컴포넌트로, 전체 레이아웃을 구성하고 Main 컴포넌트를 포함합니다.
  1. 주요 레이아웃 및 뷰 (Layouts & Views)
    • src/views/Main.js: Header, Content, Footer, AlertMsg 컴포넌트를 포함하여 애플리케이션의 기본적인 페이지 레이아웃을 정의합니다.
    • src/views/layout/Header.js: 애플리케이션의 상단 바를 담당하며, 상담원 상태 표시, 알림 (alertCall, alertTask, alertChat), 언어 변경(i18n 사용), 그리고 채널 탭(Voice, Chat, Task, Setting) 간의 전환 기능을 제공합니다.
      • toggle(): 드롭다운 메뉴를 토글합니다.
      • selectAgentState(): 상담원 상태를 변경합니다.
      • updateChannelTab(): 현재 활성화된 채널 탭을 업데이트합니다.
    • src/views/layout/Content.js: 메인 콘텐츠 영역을 정의하며, 선택된 채널 탭(Voice, Chat, Task, Setting)에 따라 다른 컴포넌트를 동적으로 렌더링합니다. Amazon Connect(CCP), Zendesk, AWS, Outbound 등 외부 서비스와의 연동 로직을 포함합니다.
      • ccpRef, zendeskRef, voiceRef, awsRef, taskRef, chatRef, settingRef, outboundRef: 하위 컴포넌트의 인스턴스를 참조하기 위한 React.createRef입니다.
      • voiceFunc(), zendeskFunc(), awsFunc(), taskFunc(), chatFunc(), settingFunc(): 각 기능별 컴포넌트의 메서드를 호출하는 헬퍼 함수입니다.
    • src/views/layout/Footer.js: (제공된 코드에서는 비어있지만) 일반적으로 애플리케이션의 하단 영역을 담당합니다.
    • src/views/layout/AlertMsg.js: 팝업 알림 메시지를 표시합니다.
    • src/views/layout/Loading.js: 로딩 스피너를 표시하여 데이터 로딩 중임을 나타냅니다.
    • src/views/layout/Login.js: 사용자 로그인 처리를 담당합니다. update_login_status 액션을 통해 로그인 상태를 업데이트합니다.
  1. 핵심 기능 모듈 (Core Functionality Modules)
    • src/libs/CCP.js: Amazon Connect Streams API와 Task.js, Chat.js 라이브러리를 사용하여 Amazon Connect CCP(Contact Control Panel)와의 상호작용을 처리합니다. 이는 애플리케이션의 가장 중요한 부분으로, 전화 걸기, 받기, 통화 제어, 상담원 상태 관리, 퀵 커넥트, 태스크 관리, 채팅 세션 관리 등 모든 통신 관련 로직을 포함합니다.
      • initConnect(): Amazon Connect CCP를 초기화하고, 다양한 Amazon Connect 이벤트 (예: contact.onConnecting, contact.onConnected, agent.onStateChange)를 구독합니다. Zendesk 사용 여부에 따라 설정값을 동적으로 로드합니다.
      • handleContactConnecting(), handleContactConnected(), handleContactEnded(), handleContactMissed(): 각각 통화 연결 중, 연결됨, 종료됨, 부재중 이벤트에 대한 핸들러입니다.
      • handleAgentStateChange(), handleAgentRoutable(): 상담원 상태 변경 이벤트를 처리하고, 상담원 상태 목록을 업데이트합니다.
      • makeCallConnect(), makeCallCisco(): Amazon Connect 또는 Cisco를 통해 전화를 겁니다.
      • acceptCall(), dropCall(): 인입 호출을 수락하거나 현재 통화를 종료합니다.
      • holdCall(), retrieveCall(): 통화를 보류하거나 보류된 통화를 재개합니다.
      • muteCall(), unmuteCall(): 통화를 음소거하거나 음소거를 해제합니다.
      • conferenceCall(), addThirdParty(): 다자간 통화를 시작하거나 기존 통화에 다른 참가자를 추가합니다.
      • getQuickList(): 퀵 커넥트 목록을 조회합니다.
      • getTaskTemplate(): 태스크 템플릿을 가져옵니다.
      • insertCallData(), updateCallData(): 통화 데이터를 백엔드 API를 통해 저장/업데이트합니다.
      • getMRP(), getOriginNumber(): 전화번호 관련 정보를 가져옵니다.
      • createTicketByCall(), updateTicketWithMent(), updateTicketWithCustomFields(): Zendesk 티켓을 생성하고 업데이트하는 함수 (Zendesk 연동 시).
      • insertLog(): 서버에 로그를 기록합니다.
      • alertMsg(): 사용자에게 알림 메시지를 표시합니다.
      • getFormattedDate(), msToTime(): 시간 관련 유틸리티 함수를 사용합니다.
    • src/libs/Zendesk.js: Zendesk Support와의 연동을 담당합니다. Zendesk App Framework (ZAF) Client를 사용하여 Zendesk API를 호출하고, 티켓 생성, 사용자 정보 조회, 통화 기록 업데이트 등의 기능을 수행합니다.
      • init(): ZAF Client를 초기화하고 Zendesk 관련 이벤트를 구독합니다.
      • getConfig(), getCurrentUserInfo(), getUserFields(): Zendesk 설정 및 사용자 정보를 가져옵니다.
      • createTicket(), updateTicket(), updateTicketWithTags(): Zendesk 티켓을 생성하고 업데이트하며, 태그를 관리합니다.
      • createUserByPhoneNumber(), updateUserInfo(), popUser(): Zendesk 사용자 정보를 생성, 업데이트, 조회합니다.
      • getTranscript(): 채팅 내용을 가져옵니다.
    • src/libs/AWS.js: AWS 서비스와의 상호작용 (구체적인 내용은 CCP.js에서 대부분 처리)을 위한 추상화 계층으로 보입니다.
    • src/libs/Outbound.js: 아웃바운드 통화 관련 로직을 담당하며, MRP(Main Receiving Number) 목록 조회, 통화 데이터 삽입/업데이트 등 백엔드 API(apiUrl, insertPath, updatePath, getMRPPath, getOriginNumberPath)와 통신합니다.
    • src/libs/zafClient.js: Zendesk App Framework (ZAF) Client의 훅(hook)을 제공하는 유틸리티 파일입니다.

로직 흐름 (Conceptual Flow)

  1. 초기화 및 로그인:
    • 애플리케이션이 시작되면 index.js를 통해 App.js가 렌더링되고, Main.js에서 CCP.js와 Zendesk.js가 초기화됩니다.
    • CCP.js의 initConnect() 함수는 Amazon Connect CCP를 초기화하고, 필요한 설정값 (예: ccpUrl, region, loginPopup 등)을 적용합니다. Zendesk 연동 여부(zendesk_use_f)에 따라 Zendesk 관련 설정도 로드합니다.
    • Login.js 컴포넌트가 로그인 상태를 관리합니다.
  1. 상담원 상태 관리:
    • CCP.js는 Amazon Connect의 agent.onStateChange 이벤트를 구독하여 상담원의 상태 변화(update_agent_state)를 Redux 스토어에 반영합니다.
    • Header.js는 Redux 스토어의 agentState를 구독하여 상담원의 현재 상태를 UI에 표시하고, 드롭다운을 통해 상담원 상태를 변경할 수 있는 기능을 제공합니다 (selectAgentState()).
  1. 통화 처리 (Voice Channel):
    • CCP.js는 contact.onConnecting, contact.onConnected, contact.onEnded, contact.onMissed 등의 이벤트를 구독하여 들어오거나 나가는 통화의 상태를 감지합니다.
    • 이벤트 발생 시, CCP.js는 update_call_info 및 update_call_list 액션을 통해 통화 관련 정보를 Redux 스토어에 업데이트합니다.
    • Voice.js 컴포넌트는 callInfo 및 callList 상태를 기반으로 UI를 렌더링하고, 사용자는 onClick_makeCall(), onClick_accept(), onClick_dropCall(), onClick_hold(), onClick_retrieve(), onClick_mute(), onClick_unMute(), onClick_conference(), onClick_addThirdParty() 등의 함수를 통해 통화를 제어합니다.
    • Zendesk.js는 통화 시작 및 종료 시 Zendesk 티켓을 생성하거나 업데이트하여 통화 기록을 연동합니다. createTicketByCall() 및 updateTicketWithVoiceComments() 함수가 이 역할을 수행합니다.
  1. 채팅 처리 (Chat Channel):
    • CCP.js는 contact.onMessage 등의 이벤트를 구독하여 채팅 메시지를 수신하고 update_chat_info, update_chat_list, add_transcript 액션을 통해 채팅 상태와 내용을 관리합니다.
    • Chat.js 컴포넌트는 채팅 메시지를 표시하고, onKeyUp_msgText() 및 onclick_sendMsg()를 통해 메시지 전송 기능을 제공합니다.
    • Zendesk.js는 채팅 내용을 Zendesk 티켓에 기록하거나(getTranscript()) 앱을 통해 Zendesk로 데이터를 전송합니다 (zendeskFunc("sendToApp", ...)).
  1. 태스크 처리 (Task Channel):
    • CCP.js는 Amazon Connect의 태스크 관련 이벤트를 구독하고 (onTaskExpiring, onTaskExpired, onTransferInitiated 등) update_task_info, update_task_list 액션을 통해 태스크 상태를 Redux 스토어에 반영합니다.
    • Task.js 컴포넌트는 태스크 목록을 표시하고, onclick_createBtn(), onClick_updateTask(), onclick_dropTask() 등을 통해 태스크를 생성, 업데이트, 삭제합니다.
  1. 설정 관리 (Setting Tab):
    • Setting.js 컴포넌트는 사용자가 미디어 장치 (마이크, 스피커, 벨소리) 및 언어 설정을 변경하고 저장할 수 있는 UI를 제공합니다.
    • onclick_refresh(): CCP.js의 getMediaDevices 함수를 호출하여 현재 시스템의 미디어 장치 목록을 가져옵니다.
    • onChange_device(): 선택된 장치(speakerId, microphoneId, ringerId)를 localStorage에 저장하고 CCP.js의 setSpeakerDevice 또는 setMicrophoneDevice 함수를 호출하여 실제 장치를 설정합니다.
    • onclick_downloadLogs(): CCP.js의 downloadLogs 함수를 호출하여 Amazon Connect 로그를 다운로드합니다.
    • onclick_logout(): CCP.js의 logout 함수를 호출하여 로그아웃합니다.

Browser & External Services

설명:

  • React Client (UI Components): App.js, Main.js, Header.js, Content.js, Footer.js, AlertMsg.js, Loading.js 및 Voice.js, Chat.js, Task.js, Setting.js와 같은 모든 뷰/탭 컴포넌트를 포함합니다. 사용자가 직접 상호작용하는 부분입니다.
  • Redux Store (Actions/Reducers): 애플리케이션의 중앙 집중식 상태 관리 시스템입니다.
    • Actions (src/actions/index.js): UI 이벤트나 비동기 작업의 결과로 상태 변경을 요청합니다 (UPDATE_AGENT_STATE, UPDATE_CALL_INFO 등).
    • Reducers (src/reducers/index.js): 액션에 따라 상태를 변경하는 순수 함수입니다.
  • Core Logic:
    • CCP.js: Amazon Connect와의 모든 상호작용을 담당하는 핵심 모듈입니다. Amazon Connect Streams API를 사용하여 통화, 채팅, 태스크 이벤트를 수신하고 제어 명령을 보냅니다. 또한, google-libphonenumber를 사용하여 전화번호를 처리합니다.
    • Zendesk.js: Zendesk Support와의 연동을 담당합니다. Zendesk App Framework (ZAF)를 통해 Zendesk의 API를 호출하여 티켓, 사용자, 태그 등을 관리하고 애플리케이션과 Zendesk 간의 데이터 동기화를 수행합니다.
    • AWS.js: AWS 서비스와의 일반적인 상호작용을 위한 추상화 계층입니다.
    • Outbound.js: 아웃바운드 통화와 관련된 특정 비즈니스 로직 및 백엔드 API와의 통신을 처리합니다.
  • Channel Specific UI Components (Voice.js, Chat.js, Task.js, Setting.js): 각 채널의 고유한 UI와 사용자 상호작용 로직을 캡슐화합니다. 이들은 CCP.js, Zendesk.js 등으로부터 콜백 함수를 받아 통신 기능을 수행합니다.
  • Utilities (common.js, audioUtils.js, ring.js, ringerData.js, ccp.util.js, hooks.js): 애플리케이션 전반에서 사용되는 공통 함수, 오디오 처리, 링거 데이터, CCP 관련 유틸리티, React Hooks 등을 제공합니다.
  • Backend APIs (AWS, Zendesk): Amazon Connect 및 Zendesk의 공식 API 외에, Outbound.js에서 사용하는 connect-api.gsclip.com과 같은 커스텀 백엔드 API도 존재합니다. 이들은 MRP 조회, 아웃바운드 매치 데이터 삽입/업데이트 등 특정 비즈니스 요구사항을 처리합니다.
  • HTML Files: 각 뷰의 정적 구조를 정의하며, style.css와 jquery-1.12.4.min.js를 링크합니다.
  • CSS Files: 애플리케이션의 전반적인 시각적 스타일을 적용합니다.
  • Font Files, Image Files: 폰트와 이미지 자산을 제공합니다.
  • Language Files: 다국어 지원을 위한 텍스트 데이터를 제공합니다.

주요 함수 및 그 로직 (Function-level Logic)

주요 컴포넌트와 유틸리티 파일에서 몇 가지 중요한 함수와 그 로직을 살펴보겠습니다:

src/libs/CCP.js (Amazon Connect 연동의 핵심)

  • initConnect():
    • Zendesk 사용 여부(zendesk_use_f)에 따라 localStorage에서 Zendesk 설정(zdConfig)을 로드하거나, 자체 환경 변수를 사용합니다.
    • window.connect.core.initCCP()를 호출하여 Amazon Connect CCP를 초기화합니다. 이때, CCP URL, 리전, 로그인 팝업 설정(loginPopup, loginPopupAutoClose), 소프트폰 설정(allowFramedSoftphone, disableRingtone) 등을 options 객체로 전달합니다.
    • window.connect.agent.onStateChange(), window.connect.contact.onConnecting(), window.connect.contact.onConnected(), window.connect.contact.onEnded(), window.connect.contact.onMissed() 등 다양한 Amazon Connect 이벤트를 구독합니다. 각 이벤트는 해당 handle 함수를 호출하여 애플리케이션 상태를 업데이트합니다.
  • handleAgentStateChange(agent):
    • 에이전트의 상태가 변경될 때 호출됩니다.
    • agent.getState().name을 통해 현재 상태 이름을 확인하고, update_agent_state 액션을 디스패치하여 Redux 스토어의 상담원 상태를 업데이트합니다.
    • agent.getAgentStates()를 호출하여 전체 상담원 상태 목록을 가져오고 update_agent_state_list 액션을 디스패치합니다.
    • "MissedCallAgent" 상태일 경우, 일정 시간 후 "Available" 상태로 자동 전환하는 로직이 포함되어 있습니다(missedToAvailable, missedSeconds 설정 사용).
  • handleContactConnecting(contact):
    • 새로운 컨택트(예: 인바운드 전화)가 연결될 때 호출됩니다.
    • contact.getQueue().name을 통해 큐 이름을 가져오고, contact.getContactId()를 통해 컨택트 ID를 가져옵니다.
    • contact.getAttributes()를 통해 컨택트 속성(attributes)을 파싱하여 Zendesk 관련 사용자 정의 필드(zdkWnt0 ~ zdkWnt10)를 추출합니다.
    • 음성 통화(contact.isInbound())인 경우 playAudio()를 호출하여 벨소리를 재생합니다.
    • update_call_info 액션을 디스패치하여 통화 정보를 Redux 스토어에 업데이트합니다.
    • Zendesk가 사용되는 경우, createTicketByCall() 함수를 호출하여 Zendesk에 티켓을 생성합니다.
  • makeCallConnect(callInfo):
    • window.connect.agent.createOutboundContact()를 사용하여 아웃바운드 통화를 시작합니다. callInfo.number와 callInfo.quickConnectId를 사용합니다.
    • 성공 시 update_call_info와 update_call_list를 업데이트합니다.
  • dropCall(channel):
    • 현재 활성화된 통화 연결을 종료합니다.
    • window.myCPP.agent.getContacts(window.connect.ContactType.VOICE)[0]를 통해 현재 통화 컨택트를 가져옵니다.
    • contact.getAgentConnection().destroy()를 호출하여 연결을 끊습니다.
    • update_call_info와 update_call_list를 초기화합니다.
  • getMRP():
    • 백엔드 API(apiUrl, getMRPPath)를 호출하여 MRP(Main Receiving Number) 목록을 가져옵니다.
    • 가져온 목록을 update_config 액션을 통해 Redux 스토어에 업데이트합니다.
  • insertLog():
    • 서버 로그 플래그(serverLogF)가 true인 경우, axios.post를 사용하여 백엔드 API (apiUrl, insertPath)에 로그를 전송합니다.

src/libs/Zendesk.js (Zendesk 연동)

  • init():
    • window.ZAFClient.init()를 통해 Zendesk App Framework 클라이언트를 초기화합니다.
    • getConfig(), getCurrentUserInfo(), getUserFields()를 비동기적으로 호출하여 Zendesk 관련 설정을 로드합니다.
    • update_login_status 액션을 디스패치하여 Zendesk 로그인 상태를 true로 설정합니다.
    • g_client.off()를 통해 이전에 등록된 Zendesk 이벤트를 해제하고, 새로운 이벤트 (click_to_call, end_call, voice.dialout, api_notification.kms_result_msg)를 구독합니다.
  • createTicket(callInfo):
    • Zendesk에 새 티켓을 생성합니다. callInfo에서 고객 정보(customerName, endpoint), 티켓 유형(callType), 제목(ticketTitle), 태그(tag_custom) 등을 추출합니다.
    • g_client.request()를 사용하여 Zendesk API에 POST 요청을 보냅니다.
    • 성공 시 update_call_info를 업데이트하고, 실패 시 alertMsg()로 알림을 띄웁니다.
  • updateTicket(ticketId, data, callInfo):
    • 기존 Zendesk 티켓을 업데이트합니다. ticketId와 업데이트할 data (예: 코멘트, 태그)를 받습니다.
    • g_client.request()를 사용하여 Zendesk API에 PUT 요청을 보냅니다.
    • 성공 시 update_call_info를 업데이트하고, 실패 시 에러를 기록합니다.
  • createUserByPhoneNumber(phoneNumber, callInfo):
    • 제공된 전화번호로 Zendesk에 새 사용자를 생성합니다.
    • g_client.request()를 사용하여 Zendesk API에 POST 요청을 보냅니다.
    • 성공 시 update_created_zdk_user 액션을 디스패치하여 생성된 사용자 정보를 Redux 스토어에 저장합니다.
  • getTranscript(chatInfo):
    • 채팅 기록을 가져와 add_transcript 및 update_transcript 액션을 디스패치하여 Redux 스토어에 업데이트합니다.

src/views/layout/tab/Voice.js (음성 통화 탭)

  • constructor():
    • 현재 날짜와 시간을 초기 상태로 설정합니다.
    • react-datepicker의 로케일을 한국어(ko)로 등록합니다.
    • 환경 변수와 localStorage에 저장된 Zendesk 설정을 기반으로 countryCode, countryNumber, coldTransferUseF, voiceRefuseUseF 등의 값을 초기화합니다.
  • onKeyUp(event, mrpList):
    • 텍스트 입력 필드에서 Enter 키가 눌렸을 때 onClick_makeCall() 함수를 호출합니다.
  • onClick_makeCall(e, mrpList):
    • 통화 유형(this.state.selectQuickItem)에 따라 CCP.js의 makeCallConnect() 또는 makeCallCisco()를 호출하여 전화를 겁니다.

src/views/layout/tab/Chat.js (채팅 탭)

  • onKeyUp_msgText(event):
    • 메시지 입력 필드에서 Enter 키가 눌리고 메시지 내용이 비어있지 않을 경우 onclick_sendMsg()를 호출합니다.
  • onclick_sendMsg(e):
    • 현재 채팅 세션(agentChatSession)이 활성 상태인지 확인하고, chatMessageSendF 설정이 true인지 확인합니다.
    • agentChatSession.sendMessage()를 호출하여 메시지를 전송하고, 성공 시 messagesEndRef를 사용하여 채팅 스크롤을 최하단으로 이동합니다.
  • scrollToBottom():
    • 새로운 메시지가 추가될 때마다 채팅창을 자동으로 스크롤하기 위해 사용됩니다.

src/views/layout/tab/Task.js (태스크 탭)

  • onclick_createBtn(e):
    • 태스크 생성 패널(createComponentF)을 토글합니다.
    • 현재 상담 상태를 확인하여 태스크 생성 패널의 높이(createHeight)를 조절합니다.
  • onClick_updateTask():
    • window.myCPP.agent.updateTask()를 호출하여 태스크를 업데이트합니다.

결론

이 애플리케이션은 React, Redux, Amazon Connect Streams API, Zendesk App Framework를 활용하여 고객 서비스 상담원들이 음성 통화, 채팅, 태스크를 효율적으로 처리할 수 있도록 돕는 웹 기반 소프트폰 솔루션입니다. 모듈화된 구조를 통해 각 기능이 독립적으로 개발 및 유지보수될 수 있으며, 외부 서비스와의 긴밀한 연동을 통해 고객 상호작용 데이터를 중앙에서 관리합니다.

제공된 HTML 파일들은 개발 및 디자인 과정에서 각 화면의 스냅샷 또는 디자인 가이드 역할을 하는 것으로 보이며, 실제 애플리케이션은 React 컴포넌트들을 통해 동적으로 UI를 구성하고 있습니다.

이러한 모듈별 기능과 상호작용을 통해 상담원들은 하나의 통합된 인터페이스에서 다양한 채널의 고객 문의를 효율적으로 관리할 수 있습니다.
