무신사 MVP 최종 미팅

1) API 전달 일정 · 내부 개발 진행

  • 현재 제공 가능한 API 목록과 문서(검색/카탈로그/가격·재고/주문상태/회원·장바구니/추천/프로모션)는 무엇인가요? Postman 컬렉션 또는 OpenAPI 스펙을 주실 수 있나요?
  • 샌드박스(또는 스테이징) 환경 URL, 인증 방식(API Key/OAuth/IP 허용목록), 요청 한도·QPS는 어떻게 되나요?
  • 검색 API: 질의 파라미터, 정렬/필터(사이즈, 색상, 카테고리, 가격대), 페이지네이션 규칙(offset/limit vs cursor), 응답 스키마(상품 id/title/images/price/재고/카테고리/속성) 확정본을 공유해 주실 수 있나요?
  • 카탈로그/재고 갱신 주기(실시간/주기배치), 가격 변동/할인/품절 이벤트의 전파 지연(SLA)은 얼마인가요?
  • 주문/배송/반품 조회 API가 1차 범위에 포함되나요? 포함 시 개인정보 마스킹 정책은?
  • 오류 코드/타임아웃/재시도 정책, 장애 시 대체 경로(캐시/디그레이드) 기준은?
  • API 전달(문서/키/테스트 계정) 목표일자와 내부 개발 완료 예상일을 알려주실 수 있을까요?

2) 데이터셋 전달 일정 (9/19 요청분 회신)

  • 9/19에 요청드린 데이터셋(샘플 발화, FAQ, 상품 카테고리/속성 사전, 과거 검색 로그 Top N, 품절/재입고 이벤트 샘플) 전달 가능 시점은 언제인가요?
  • 제공 형식(CSV/JSON/Parquet)과 스키마 정의, 필드 설명서(딕셔너리)를 함께 주실 수 있나요?
  • 상품 카테고리 트리/태그/동의어 사전(예: “트랙팬츠=조거팬츠”)을 제공해 주실 수 있나요?
  • 이미지 URL 사용 정책(만료/서명 URL 여부), 썸네일/대표 이미지 규칙은?
  • 개인정보/저작권/보안 이슈가 있는 필드는 제거 또는 마스킹해서 제공 가능한가요?
  • 정기 동기화 주기(일배치/시간단위/실시간)와 증분 방식(전체/증분플래그/업데이트 타임스탬프)은?

3) 착수 시작 예정일 · 마일스톤

  • 공식 착수일과 1차 알파/베타/런치 마일스톤(기능 범위)을 합의할 수 있을까요?
  • RACI(무신사/당사 담당자·승인자)와 주간 정기 미팅 일정, 이슈 트래킹 채널(Jira/Slack/메일)은?
  • 채널 범위(웹/앱/상담센터/카카오톡 등) 우선순위와 채널별 커스텀 요구사항은?
  • 배포 프로세스(스테이징 → 검수 → 운영), 변경 관리/롤백 계획, 로그·모니터링(Cloud Logging/GA4/Amplitude/Looker) 도구는?

4) 성공 지표(검수 항목 일부 제외 요청 관련)

  • 기준선(Baseline)은 무엇이며, 개선 목표(예: 무응답률 20%→10%)와 측정 기간은?
  • CS팀/MD팀과의 정성 지표(CSAT/NPS/상담사 피드백) 수집 방식은?

  • 우선 지원 언어(ko/en)와 지원할 언어 범위는?

  • 실패/무응답 시 휴먼 핸드오버 기준(상담 연결 조건, 운영 시간, 연결 채널)은?

프론트/웹훅 계약 (현재 PoC 구조 반영)

  • 무신사 프론트에서 필요한 추가 필드(예: 배송비, 쿠폰/프로모션, 리뷰 요약, 사이즈 추천)가 있나요?
  • 웹훅 보안(헤더 시크릿/Basic Auth/IP 허용)과 타임아웃·재시도·멱등 키(responseId+session) 정책 합의가 필요합니다. 기준은?
  • 이미지/리치카드 UI 스펙(1행 N개, 캐러셀/그리드, 필수 필드)과 접속 채널별 제약은?
