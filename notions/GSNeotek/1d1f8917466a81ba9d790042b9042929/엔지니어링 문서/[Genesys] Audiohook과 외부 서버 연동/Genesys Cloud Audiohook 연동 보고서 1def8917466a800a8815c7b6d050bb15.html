Genesys Cloud Audiohook 연동 보고서

1. 개요

본 보고서는 Genesys Cloud AudioHook과의 성공적인 WebSocket 통신 구현에 관한 기술적 내용을 담고 있습니다. AudioHook은 Genesys Cloud의 음성 데이터를 실시간으로 외부 시스템에 스트리밍하여 음성 인식(STT), 감정 분석 등 다양한 음성 처리 서비스를 구현할 수 있게 해주는 인터페이스입니다.

LG CNS에서 요청한 EU AICC 베트남 STT 연계 작업에 앞서, Genesys Cloud AudioHook과의 기본적인 통신 방법과 구현 방안을 설명합니다.

2. AudioHook 아키텍처 및 개념

2.1 AudioHook 개요

AudioHook은 Genesys Cloud에서 음성 데이터를 외부 시스템으로 전달하기 위한 WebSocket 기반 프로토콜입니다. 이 기능을 통해 다음과 같은 작업이 가능합니다:

  • 실시간 콜 센터 통화 오디오 스트리밍
  • 외부 음성 인식(STT) 서비스와의 연동
  • 감정 분석, 키워드 감지 등 고급 음성 분석 기능 구현

2.2 통신 구조

AudioHook은 다음과 같은 통신 구조를 갖습니다:

  1. WebSocket 연결 수립: Genesys Cloud는 지정된 엔드포인트로 WebSocket 연결을 시도합니다.
  1. 세션 시작: Genesys Cloud가 open 메시지를 전송하고, 서버는 opened 메시지로 응답합니다.
  1. 오디오 스트리밍: 통화 중 발생하는 오디오 데이터가 바이너리 형태로 전송됩니다.
  1. 세션 종료: Genesys Cloud가 close 메시지를 전송하고, 서버는 closed 메시지로 응답합니다.

2.3 AudioHook 연동 API 종류

1. AudioHook 설정 및 관리 API

  • Integration Configuration API
    • 목적: AudioHook 통합을 생성하고 관리하는 API
      • 주요 메서드: POST, GET, PUT, DELETE /api/v2/integrations
      • 설명: AudioHook 통합을 생성하고 설정하는 데 사용됩니다. 여기서 웹소켓 엔드포인트 URL, 인증 방법, 미디어 설정 등을 구성합니다.
    • Credential Management API
      • 목적: AudioHook 통합에 필요한 인증 정보 관리
      • 주요 메서드: POST, GET, PUT /api/v2/integrations/{integrationId}/credentials
      • 설명: OAuth 클라이언트 ID/시크릿, API 키 등의 인증 정보를 관리합니다.

    2. WebSocket 프로토콜 API

    AudioHook은 WebSocket 프로토콜을 통해 통신하며, 다음과 같은 주요 메시지 유형을 지원합니다:

    • 세션 제어 메시지
      • open: 클라이언트(Genesys)에서 세션 시작 요청
      • opened: 서버(개발자 구현)에서 세션 시작 응답
      • close: 클라이언트에서 세션 종료 요청
      • closed: 서버에서 세션 종료 응답
      • ping/pong: 연결 상태 확인용 메시지
      • error: 오류 보고 메시지
      • disconnect: 서버에서 세션 강제 종료 요청
    • 상태 제어 메시지
      • pause: 오디오 스트리밍 일시 중지 요청
      • paused: 일시 중지 확인 응답
      • resume: 오디오 스트리밍 재개 요청
      • resumed: 재개 확인 응답
      • update: 세션 매개변수 업데이트 요청
      • updated: 업데이트 확인 응답
    • 오디오 데이터 전송
      • 바이너리 메시지: PCM 또는 기타 포맷의 오디오 데이터 전송
      • discarded: 오디오 데이터 손실 알림 메시지

    3. 통화 관리 및 제어 API

    • Conversation API
      • 목적: 통화 세션 정보 조회 및 관리
      • 주요 메서드: GET /api/v2/conversations/{conversationId}
      • 설명: AudioHook에 연결된 통화의 메타데이터, 참가자 정보, 상태 등을 조회할 수 있습니다.
    • Recording API
      • 목적: 통화 녹음 관리
      • 주요 메서드: GET, PUT /api/v2/conversations/{conversationId}/recordings
      • 설명: 통화 녹음을 관리하고 액세스할 수 있습니다. AudioHook 결과를 녹음과 연계할 때 유용합니다.

    4. 분석 및 피드백 API

    • Analytics API
      • 목적: AudioHook으로 생성된 데이터 분석 및 보고
      • 주요 메서드: POST /api/v2/analytics/conversations/details/query
      • 설명: AudioHook을 통해 처리된 통화 데이터를 분석하고 보고서를 생성할 수 있습니다.
    • Attributes API
      • 목적: 통화에 커스텀 속성 추가
      • 주요 메서드: PATCH /api/v2/conversations/{conversationId}/attributes
      • 설명: STT 결과나 감정 분석 결과 등을 통화 속성으로 추가할 수 있습니다.

6. 시퀀스 관리 중요사항

  • 시퀀스 번호 관리가 매우 중요합니다. 각 메시지에는:
    • seq: 발신자 메시지 번호 (증가하는 값)
    • clientseq: 서버가 마지막으로 받은 클라이언트 메시지 번호
    • serverseq: 클라이언트가 마지막으로 받은 서버 메시지 번호
  • 시퀀스 불일치는 통신 실패의 주요 원인이 되므로 정확히 관리해야 합니다.

Genesys Cloud AudioHook과 성공적으로 연동하려면 위의 API들을 정확히 이해하고 구현해야 합니다. 특히 WebSocket 프로토콜과 시퀀스 번호 관리가 가장 중요한 부분입니다.

3. WebSocket 서버 구현 및 통신 프로토콜

3.1 기본 서버 구현

Node.js와 Express 프레임워크를 사용하여 WebSocket 서버를 구현했습니다:

const express = require("express");
const http = require("http");
const WebSocket = require("ws");

const app = express();
const server = http.createServer(app);
const wss = new WebSocket.Server({ server });

// 서버 구현 코드
// ...

const PORT = process.env.PORT || 8080;
server.listen(PORT, () => {
  console.log(`WebSocket 서버 실행 중: ws://localhost:${PORT}`);
});

3.2 메시지 유형 및 처리

AudioHook 프로토콜에서 지원하는 주요 메시지 유형은 다음과 같습니다:

  • open/opened: 세션 시작 및 응답
  • ping/pong: 연결 유지 및 네트워크 상태 확인
  • close/closed: 세션 종료 및 응답
  • error: 오류 발생 알림

3.3 시퀀스 번호 관리

AudioHook 프로토콜에서는 메시지 순서와 동기화를 보장하기 위해 시퀀스 번호 관리가 매우 중요합니다:

  • seq: 각 메시지의 순차적 번호로, 발신자가 증가시킵니다.
  • clientseq: 서버가 마지막으로 받은 클라이언트 메시지의 시퀀스 번호입니다.
  • serverseq: 클라이언트가 마지막으로 받은 서버 메시지의 시퀀스 번호입니다.

올바른 시퀀스 번호 관리가 이루어지지 않으면 "메시지가 잘못된 형식"이라는 오류나 "잘못된 서버 시퀀스 번호" 오류가 발생합니다.

4. 구현 과정 및 트러블슈팅

4.1 초기 서버 구현

초기 서버 구현에서는 Genesys Cloud가 기대하는 정확한 메시지 형식과 시퀀스 번호 관리 방식을 파악하지 못해 연결 프로브 테스트에 실패했습니다.

� 메시지 수신: error {
  "version": "2",
  "id": "...",
  "type": "error",
  "seq": 2,
  "position": "PT0S",
  "parameters": {
    "code": 400,
    "message": "Connection probe error: Message is malformed"
  },
  "serverseq": 0
}

4.2 문제 분석 및 해결

주요 문제점과 해결 방안:

  1. 잘못된 메시지 구조: Genesys Cloud가 기대하는 정확한 메시지 형식을 파악하고 구현했습니다.
  1. 시퀀스 번호 관리 오류: AudioHook 프로토콜 문서를 분석하여 정확한 시퀀스 번호 관리 방식을 구현했습니다.
  1. 필드 누락 또는 과다: 필수 필드만 포함하도록 메시지 구조를 단순화했습니다.

4.3 최종 구현

성공적인 통신을 위한 핵심 코드 부분:

ws.on("message", (data, isBinary) => {
  if (isBinary) {
    console.log("오디오 수신:", data.length, "bytes");
    return;
  }

  try {
    const msg = JSON.parse(data.toString());

    // 클라이언트 시퀀스 번호 업데이트
    if (msg.seq !== undefined) {
      lastClientSeq = msg.seq;
    }

    switch (msg.type) {
      case "open": {
        // 응답 생성
        const openedMsg = {
          "version": "2",
          "type": "opened",
          "id": msg.id,
          "seq": serverSeq,
          "clientseq": lastClientSeq,
          "parameters": {
            "startPaused": false,
            "media": [
              // 미디어 옵션 설정
            ]
          }
        };

        ws.send(JSON.stringify(openedMsg));
        serverSeq++; // 서버 시퀀스 증가
        break;
      }

      // 다른 메시지 타입 처리...
    }
  } catch (err) {
    console.error("JSON 파싱 실패:", err.message);
  }
});

6. EU AICC STT 연계를 위한 권장 구현 방법

6.1 기본 아키텍처

EU AICC와 Genesys Cloud를 연계하기 위한 기본 아키텍처는 다음과 같습니다:

  1. AudioHook 서버: Genesys Cloud의 오디오 데이터를 수신하는 WebSocket 서버
  1. 오디오 처리 모듈: 수신된 오디오를 STT 서비스가 처리할 수 있는 형식으로 변환
  1. STT 연동 모듈: 베트남 STT 서비스와 통신하여 음성을 텍스트로 변환
  1. 결과 처리 모듈: 변환된 텍스트 데이터를 활용하는 비즈니스 로직

6.2 필요한 API 연동

EU AICC의 베트남 STT 연계를 위해 필요한 API 연동은 다음과 같습니다:

  1. Genesys AudioHook API: WebSocket을 통한 오디오 스트리밍 수신
  1. 베트남 STT API: 음성 데이터를 텍스트로 변환하는 API
  1. 결과 저장/활용 API: STT 결과를 저장하거나 활용하는 API

6.3 개발 공수 산정을 위한 고려사항

EU AICC에서 개발 공수를 산정할 때 고려해야 할 사항:

  1. WebSocket 서버 구축: AudioHook 프로토콜을 준수하는 서버 개발
  1. 오디오 데이터 처리: PCM 포맷의 오디오 데이터 처리 로직 개발
  1. STT 서비스 연동: 베트남 STT 서비스와의 연동 개발
  1. 결과 처리 및 저장: 변환된 텍스트 데이터 처리 및 저장 기능 개발
  1. 모니터링 및 로깅: 시스템 모니터링 및 로그 관리 기능 개발
  1. 성능 및 확장성: 동시 호출 처리 능력 및 확장성 고려

7. 결론 및 제언

Genesys Cloud AudioHook과의 성공적인 연동을 위해서는 AudioHook 프로토콜의 정확한 이해와 구현이 필수적입니다. 특히 메시지 형식과 시퀀스 번호 관리가 매우 중요합니다.

EU AICC와 베트남 STT 연계를 위해서는 다음 사항을 권장합니다:

  1. AudioHook 프로토콜 기반의 안정적인 WebSocket 서버 구현
  1. 오디오 데이터 처리 및 STT 변환을 위한 모듈 개발
  1. 실시간 처리를 위한 성능 최적화
  1. 장애 상황 대응을 위한 모니터링 체계 구축

본 구현을 기반으로 EU AICC와 베트남 STT 연계를 진행한다면, 실시간 음성 인식 기능을 성공적으로 구현할 수 있을 것입니다.
