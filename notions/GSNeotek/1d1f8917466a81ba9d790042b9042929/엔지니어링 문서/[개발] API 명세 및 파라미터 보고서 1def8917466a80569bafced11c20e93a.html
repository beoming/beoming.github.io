API 명세서

1. 부서 정보 관리 API

1.1. 부서 목록 조회

  • URL: /api/cache/dept
  • Method: GET
  • Description: 캐시된 모든 부서 목록을 조회합니다.
  • Auth Required: No
  • Request: None
  • Response 예시:
    {  "success": true,  "message": "부서 목록 조회 성공",  "data": [    {      "deptcode": "D001",      "deptname": "경영지원부",      "uppercode": null,      "sortno": 1    },    {      "deptcode": "D002",      "deptname": "개발부",      "uppercode": null,      "sortno": 2    }  ],  "count": 2}

1.2. 부서 코드로 부서 조회

  • URL: /api/cache/dept/search/code
  • Method: POST
  • Description: 부서 코드를 기준으로 특정 부서를 조회합니다.
  • Auth Required: No
  • Request Parameters:
    파라미터명타입필수 여부설명
    deptCodeString필수부서 코드
  • Request Body 예시:
    {
  "deptCode": "D001"
}
  • Response 예시:
    {
  "success": true,
  "message": "부서 조회 성공",
  "data": {
    "deptcode": "D001",
    "deptname": "경영지원부",
    "uppercode": null,
    "sortno": 1
  }
}

1.3. 부서 목록 캐시 갱신

  • URL: /api/cache/dept/refresh
  • Method: POST
  • Description: 부서 목록 캐시를 강제로 갱신합니다.
  • Auth Required: No
  • Request: None
  • Response 예시:
    {  "success": true,  "message": "부서 목록 캐시 갱신 성공",  "count": 42}

1.4. 부서명으로 부서 검색

  • URL: /api/cache/dept/search
  • Method: GET
  • Description: 부서명에 특정 키워드가 포함된 부서들을 검색합니다.
  • Auth Required: No
  • Request Parameters:
    파라미터명타입필수 여부설명
    keywordString필수검색할 부서명 키워드
  • Request 예시: /api/cache/dept/search?keyword=경영
  • Response 예시:
    {
  "success": true,
  "message": "부서 검색 성공",
  "data": [
    {
      "deptcode": "D001",
      "deptname": "경영지원부",
      "uppercode": null,
      "sortno": 1
    },
    {
      "deptcode": "D005",
      "deptname": "경영기획팀",
      "uppercode": "D001",
      "sortno": 1
    }
  ],
  "count": 2
}

2. 인바운드 콜 이벤트 API

2.1. 인바운드 콜 이벤트 수신

  • URL: /api/inbound/call
  • Method: POST
  • Description: 상담사 법인폰으로 인바운드 전화가 왔을 때 이벤트를 수신합니다.
  • Auth Required: No
  • Request Parameters:
    파라미터명타입필수 여부설명
    callerPhoneNumberString필수발신자(고객) 전화번호

    {
  "callerPhoneNumber": "010-1234-5678",
  "agentPhoneNumber": "010-9876-5432",
  "callId": "call-20250515-001"
}
  • Response 예시:

  "notified": true,

    "cgempno": "1010123",

    "telExchNo": "1234",
    "telSeq": "5678",
    "inskdGrupLccd": "01",
    "mobilCallJobLccd": "A",
    "mobilCallJobMccd": "A1",

    "mobilCallTpcd": "I",

4. 웹소켓 클라이언트 예시 (소프트폰 연결)

JavaScript (소프트폰 클라이언트)

// StompJS 라이브러리 사용 예시
const stompClient = new StompJs.Client({
  brokerURL: 'ws://localhost:8080/ws',
  debug: function (str) {
    console.log(str);
  },
  reconnectDelay: 5000
});

// 소프트폰 연결 성공 시
stompClient.onConnect = function (frame) {
  console.log('소프트폰 웹소켓 연결 성공: ' + frame);

  // 상담사 ID로 주제 구독
  const agentId = 'agent001'; // 실제 상담사 ID로 대체
  stompClient.subscribe('/topic/' + agentId, function (message) {
    // 인바운드 콜 알림 수신
    const notification = JSON.parse(message.body);
    console.log('인바운드 콜 알림 수신:', notification);

    // 알림 UI 표시 (예시)
    if (notification.eventType === 'INBOUND_CALL') {
      // 고객 정보 추출
      const customerInfo = notification.customerInfo;
      const callerName = customerInfo.customerName || '알 수 없음';
      const callerPhone = notification.callerPhoneNumber;

      // 인바운드 콜 팝업 표시
      showInboundCallPopup({
        callerName: callerName,
        callerPhone: callerPhone,
        callId: notification.callId,
        customerInfo: customerInfo
      });
    }
  });
};

// 연결 오류 발생 시
stompClient.onStompError = function (frame) {
  console.error('소프트폰 웹소켓 오류: ' + frame.headers['message']);
  console.error('추가 정보: ' + frame.body);
};

// 웹소켓 연결 시작
stompClient.activate();

// 인바운드 콜 팝업 표시 함수 (예시)
function showInboundCallPopup(callData) {
  // 실제 소프트폰 UI에 맞게 구현
  alert(`인바운드 콜: ${callData.callerName} (${callData.callerPhone})`);

  // 더 상세한 정보 표시 (예: 모달 다이얼로그)
  const popupElement = document.getElementById('inbound-call-popup');
  if (popupElement) {
    document.getElementById('caller-name').textContent = callData.callerName;
    document.getElementById('caller-phone').textContent = callData.callerPhone;

    // 고객 정보가 있으면 추가 정보 표시
    if (callData.customerInfo.receiptNo) {
      document.getElementById('receipt-no').textContent = callData.customerInfo.receiptNo;
    }

    // 팝업 표시
    popupElement.style.display = 'block';
  }
}

6. 참고 사항

  1. 로컬 테스트 환경:
    • 서버 주소: http://localhost:8080
    • 웹소켓 주소: ws://localhost:8080/ws
  1. 인바운드 콜 API 처리 흐름:
    • 인바운드 콜 이벤트 수신
    • 상담사 정보 조회 (법인폰 번호로)
    • 기간계 인터페이스 호출 (발신자 전화번호로)
    • 고객 정보 추출
    • 웹소켓으로 소프트폰에 알림 전송
    • 처리 결과 반환
  1. 캐시 동작 방식:
    • 최초 조회 시 DB에서 데이터를 가져와 캐시에 저장
    • 이후 조회 시 캐시에서 데이터 반환
    • 강제 갱신 API로 캐시 새로고침 가능
  1. 보안 고려사항:
    • 필요에 따라 API 인증 추가 필요
    • 운영 환경에서는 HTTPS 및 WSS 프로토콜 사용 권장
  1. 성능 최적화:
    • 부서 목록이 큰 경우 페이징 처리 고려
    • 캐시 만료 시간 및 자동 갱신 정책 설정 필요

이 API 명세서는 현재 개발된 서비스와 컨트롤러의 기능을 상세히 설명합니다. 운영 환경에 배포 시 필요에 따라 인증, 로깅, 모니터링 등의 추가 기능을 고려하세요.
