1. Zendesk QA - 통화 소스 연결 설정

이 화면은 Zendesk QA에서 외부 채널(여기서는 Amazon Connect)의 통화 데이터를 Zendesk QA로 동기화하기 위한 연결 설정 페이지입니다.

Zendesk 공식 문서 용어로는:

QA 시스템이 평가할 통화 및 대화를 가져오기 위한 “Source Connection” 설정

즉,

  • Amazon Connect → Zendesk QA로
  • 통화 녹취 + 상담 메타데이터를 자동으로 가져오는 파이프라인 설정입니다.

3. 모든 Zendesk 티켓 자동 동기화 안내

모든 Zendesk 티켓이 자동으로 동기화됩니다. 선택한 티켓을 제외하도록 선택할 수 있습니다.

이 의미는:

  • QA 평가 대상으로 Zendesk 티켓이 기본적으로 전체 포함되며
  • 조건(태그, 그룹 등)을 통해 필터링 가능하다는 뜻입니다.

4. 민감한 데이터 필드 숨기기

여기서 중요한 QA 보안 설정이 등장합니다.

민감한 필드 이름

Zendesk 티켓 필드 중 민감 정보(주민번호, 카드번호 등)를 QA 화면에서 숨기기 위해 사용됩니다.

공식 문서 기준:

  • 평가자에게 노출되면 안 되는 필드를 마스킹 처리

5. 보유 기간 설정

보유 기간: 1년

QA에 저장되는 평가 대상 통화 데이터의 보관 기간입니다.

공식 기능 의미:

  • 해당 기간이 지나면 통화 데이터는 삭제되지만
  • 평가 결과 리포트는 유지될 수 있음

6. 고급 옵션 (중요)

여기가 QA 보안 및 데이터 정책의 핵심입니다.

① 고객 데이터 마스킹

Zendesk 공식 설명:

최종 사용자 이름, 이메일 및 전화번호를 마스킹

즉:

  • QA 평가자가 고객 실명을 보지 못하게 보호

② 대화 콘텐츠 무시

대화 내용을 저장하지 않음

QA 평가 시:

  • 녹취는 있지만 텍스트 전사나 메시지 내용은 저장 안 함

③ 첨부 파일 무시

첨부 파일 URL 저장 안 함

통화 중 공유된 파일을 QA 시스템에 저장하지 않게 합니다.

8. Amazon Connect 연동 영역 (QA 자동 수집 핵심)

이 부분이 자동 QA와 직접 연결되는 핵심 시스템 연동 구간입니다.

🔹 Amazon Connect 추가 (토글 ON)

Amazon Connect에서 통화 정보를 자동으로 수집하겠다는 의미입니다.

입력 필드 설명:

항목설명
AWS 액세스 키 IDAmazon Connect 데이터를 읽기 위한 IAM 계정
AWS 비밀 액세스 키해당 계정의 인증 비밀키
AWS 리전Amazon Connect가 운영 중인 리전
통화 녹음 저장 위치S3 내 CallRecordings 경로

Zendesk QA는 이 정보를 통해:

  1. Amazon S3 녹취 파일 위치 조회
  1. 통화 메타데이터 연결
  1. 자동 QA 분석 및 평가 준비

9. 이 설정과 QA / 자동 QA의 관계

이 연결 설정은 다음과 같은 역할을 합니다.

일반 QA

  • 상담사가 진행한 통화가 QA 리스트로 들어옴
  • 평가자가 직접 점수 및 코멘트 입력

자동 QA (Auto QA)

  • AI가 자동으로 통화를 분석
  • 스크립트 준수율, 감정, 키워드 등을 평가

→ 이 모든 데이터의 출발점이 바로 “이 연결 설정”입니다.
