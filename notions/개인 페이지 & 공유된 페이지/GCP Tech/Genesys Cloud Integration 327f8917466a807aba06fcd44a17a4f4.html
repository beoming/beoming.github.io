Genesys Cloud Integration

Genesys Cloud Agent Desktop 안에 Google Cloud Agent Assist UI를 통합해서 상담사가 채팅 중에 AI 기능을 바로 사용할 수 있게 하는 것.

지원 방식

  • 하나의 통합 패널(Container) 안에 여러 AI 기능을 넣는 방식
  • 필요한 기능만 개별 모듈로 붙이는 방식

제공되는 주요 UI 기능

Agent Assist는 미리 만들어진 UI 모듈을 제공합니다.

1. Container

  • 여러 Agent Assist 기능을 한 패널에 모아 보여주는 메인 컨테이너

2. Generative Knowledge Assist

  • 대화 맥락에 맞는 지식 문서 추천

3. Proactive Generative Knowledge Assist

  • 문서 추천을 넘어서
  • 대화 내용을 바탕으로 AI 응답 초안까지 생성

4. Smart Reply

  • 자주 쓰는 답변을 빠르게 추천

5. Conversation Summarization

  • 상담 요약 자동 생성

6. Live Transcription

  • 대화 기록/실시간 대화 내용 표시

7. Sentiment Analysis

  • 고객 감정 분석

전체 통합 절차

1) 애플리케이션 서버 배포

먼저 UI를 렌더링할 애플리케이션 서버를 배포합니다.

역할:

  • Agent Assist UI 화면 제공
  • iFrame으로 Genesys Cloud 안에 삽입될 화면 제공

2) 백엔드 서버 배포

다음으로 백엔드 서버를 배포합니다.

역할:

  • API 호출 처리
  • 인증 처리
  • 데이터 관리
  • 프록시 역할

즉, 프론트 UI만 띄우는 게 아니라 뒤에서 API와 인증을 처리할 서버도 필요합니다.

3) Genesys Cloud Interaction Widget 설정

Genesys Cloud에서 Interaction Widget을 설정해야 합니다.

이 위젯 안에 Agent Assist UI를 iFrame 형태로 붙이게 됩니다.

여기서 중요한 건:

  • 위젯 URL
  • 동적 파라미터(conversation ID, host origin 등)
  • iFrame sandbox/permission 설정

입니다.

OAuth Client 설정

이 단계가 핵심입니다.

Genesys Cloud와 Agent Assist 앱이 안전하게 인증하려면

OAuth Client를 만들어야 합니다.

필수 설정 포인트

Grant Type

반드시: Implicit Grant (Browser)

또는 설명대로

Token Implicit Grant 방식

으로 설정해야 합니다.

이건 브라우저 기반 앱에서 로그인 후 토큰을 받기 위한 방식입니다.

Authorized redirect URIs

인증 완료 후 Genesys Cloud가 사용자를 돌려보낼 URL을 등록해야 합니다.

즉:

  • Agent Assist 앱 URL
  • redirect 처리 URL

을 정확히 넣어야 합니다.

Token Duration

토큰 만료 시간은:

3600초 (1시간)

로 설정합니다.

이유:

  • Dialogflow API access token도 1시간 기준이라
  • 만료 시간을 맞춰두는 게 운영상 편리함

생성 후 필요한 값

OAuth Client 생성 후에는 Client ID를 복사해서

애플리케이션 서버 환경 변수에 넣어야 합니다.

예:

  • OAUTH_CLIENT_ID
  • REDIRECT_URI

그리고 환경 변수 반영 후 서버를 다시 배포해야 합니다.

Messenger 권장

문서에서는 Chat V2보다 Messenger 사용을 권장합니다.

이유:

  • 기능이 더 풍부함
  • 최신 기능과 확장성이 더 좋음

즉, 새로 구성한다면 가능하면 Genesys Cloud Messenger 기반으로 가는 게 좋다는 뜻입니다.

애플리케이션 서버 환경 변수

앱 서버에는 최소한 아래 정보들이 들어갑니다.

  • OAuth Client ID
  • Genesys Cloud Region
  • Proxy Server URL

즉, 서버는:

  • 어느 Genesys 리전에 붙을지
  • 어떤 OAuth client를 쓸지
  • 어떤 백엔드/프록시와 통신할지

알고 있어야 합니다.

Interaction Widget URL 설정

Interaction Widget URL에는 정적인 URL만 넣는 게 아니라

동적 값 placeholder를 포함해야 합니다.

예시 개념:

  • conversation ID
  • host origin
  • 기타 현재 세션 관련 값

이렇게 해야 상담 세션마다 올바른 conversation context를 받아서

Agent Assist가 맞는 채팅에 연결됩니다.

iFrame 관련 설정

Genesys Cloud 안에 Agent Assist UI를 iFrame으로 넣기 때문에

브라우저 보안 설정이 중요합니다.

조정해야 하는 항목:

  • Iframe Sandbox Options
  • Iframe Feature and Permissions Policies

이걸 적절히 안 열어주면:

  • 인증 팝업
  • 스크립트 실행
  • API 호출
  • 임베드 기능

등이 정상 동작하지 않을 수 있습니다.

테스트 방법

통합 후에는 반드시 테스트해야 합니다.

테스트 포인트

  • Genesys Cloud developer tools로 테스트
  • Web Messenger / Web Chat V2 동작 확인
  • Agent가 On Queue 상태인지 확인
    • 그래야 테스트 채팅이 실제로 상담사에게 들어옴

즉, UI만 잘 떠도 끝이 아니라:

  • 상담 유입
  • 위젯 로딩
  • OAuth 인증
  • Agent Assist 패널 표시
  • 추천/요약/코칭 기능 동작

까지 확인해야 합니다.

최종적으로 보이게 되는 모습

구성이 끝나면 Genesys Cloud Agent Desktop 안에서:

  • Agent Coaching
  • Proactive Generative Knowledge Assist
  • 각종 AI 추천 기능

이 상담 화면 옆 패널 또는 통합 패널 안에서 보이게 됩니다.

즉 상담사는 기존 Genesys 화면을 크게 벗어나지 않고

같은 데스크톱 안에서 AI 기능을 바로 활용할 수 있습니다
