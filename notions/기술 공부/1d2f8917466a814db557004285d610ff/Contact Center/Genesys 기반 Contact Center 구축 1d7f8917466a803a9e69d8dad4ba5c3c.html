Genesys 기반 Contact Center 구축

Genesys Cloud를 활용한 Contact Center 구축 → 운영 → 관리 전 과정에 필요한 핵심 기술 요소와 주요 용어

기술 & 용어 정리

🏗️ 1. 구축 (Build / Setup)

💡 핵심 기술

항목설명
SIP 트렁크 설정BYOC(BYO Carrier or Premises) 설정 시 필요. PSTN 또는 SIP망 연결
Edge 장비 구성BYOC Premises일 경우 SIP, RTP를 중계하는 로컬 장비
WebRTC 전화 설정브라우저 기반 상담사 전화 설정. 별도 장비 없이 통화
전화번호(DID) 설정인바운드/아웃바운드 번호 등록 및 라우팅 연결
IVR(Call Flow)Architect로 통화 흐름 설계 (음성메뉴, 조건 분기, 대기음 등)
Queue 설정대기열 생성 및 상담사 연결 조건 정의
스킬 기반 라우팅상담사 스킬 등록 및 고객 요청 매칭
스케줄 / 업무시간 설정근무시간, 공휴일, 비상상황 라우팅 관리

📘 주요 용어

용어설명
BYOC (Bring Your Own Carrier)사용자가 자체 SIP 트렁크를 사용하여 Genesys Cloud와 연결
EdgeSIP 트래픽을 중계하는 Genesys 전용 장비 (On-Premises 시 필요)
DID (Direct Inward Dialing)외부 고객이 직접 연결할 수 있는 전화번호
ArchitectGenesys의 IVR 및 콜 플로우 설계 도구
FlowArchitect에서 설계하는 IVR 흐름 단위 (Inbound, Outbound 등)
TDM(Time Division Multiplexing)여러 개의 디지털 또는 아날로그 신호를 시간을 나누어 하나의 채널로 전송하는 방식

⚙️ 2. 운영 (Run / Launch)

💡 운영 중 필요한 기술

항목설명
Agent UI 구성상담사에게 할당된 Interaction 처리 화면
스크린 팝(Screen Pop)고객 전화 시 CRM 정보 자동 팝업
CTI 연동전화와 컴퓨터 시스템 통합. CRM, ERP 연동 필수
Call Recording / STT통화 녹취 및 음성 인식 데이터 저장
ACD (자동 콜 분배)고객 콜을 가장 적합한 상담사에게 자동 연결
Alert / Notification트래픽 급증, 장애 발생 시 관리자 알림
Omni-channel 처리음성 외 채널 (채팅, 이메일, 메신저 등) 통합 운영

📘 주요 용어

용어설명
WebRTC브라우저 내 실시간 음성/영상 통신 기술
ACD (Automatic Call Distributor)콜을 자동 분배해주는 엔진
Interaction고객과의 통화, 채팅, 메시지 등 모든 상호작용 단위
Presence / Routing Status상담사의 상태 (Available, On Queue, Busy 등)
Wrap-up Code상담 종료 후 분류하는 결과 코드

🛠️ 3. 관리 (Manage / Monitor)

💡 관리에 필요한 기술

항목설명
Performance Dashboard대시보드를 통해 큐, 에이전트, Interaction 실시간 모니터링
보고서(Analytics)히스토리, 상담시간, 대기시간, 포기율 등 정량 분석
Workforce Management(WFM)인력 스케줄링, 근무 예측, 이탈율 분석 등
Quality Management(QM)상담 품질 관리. 통화 평가, 피드백, 코칭 기록
Speech & Text AnalyticsSTT 기반 상담 분석 (감정, 키워드 등)
Routing 변경 관리Architect에서 라우팅 수정 및 배포 이력 관리
권한/역할(Role)사용자마다 접근 가능 메뉴 및 기능 설정

📘 주요 용어

용어설명
WEM (Workforce Engagement Management)WFM + QM + STT를 포함한 상담사 성과 관리 모듈
Divisions / Groups조직의 단위별로 자원 분할 (권한, 라우팅 등)
Role / Permission사용자 접근 권한 관리
Flow VersioningArchitect에서 흐름의 버전 관리
Predictive Engagement실시간 고객 행동 기반 상담사 개입 추천 (AI 활용)

🌐 기타 통합 기술 요소

요소설명
REST API / Webhooks외부 시스템과 연동 (CRM, ERP, Slack 등)
Data Actions외부 API 호출을 Flow에 삽입 가능
Lambda IntegrationAWS Lambda와 통합하여 동적 라우팅 구현
Single Sign-On (SSO)SAML 기반 로그인 연동
SCIM / AD 연동사용자 계정 동기화 (Azure AD 등)

✅ 정리: Genesys Cloud Contact Center 구성 3단계

1. 구축 (Build) → SIP, Edge, DID, Architect, Queue
2. 운영 (Run) → Agent UI, Routing, CTI, Recording, WebRTC
3. 관리 (Manage) → Dashboard, Analytics, WFM, QM, API
