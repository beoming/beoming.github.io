1. Customer (고객)

  • 실제 전화를 거는 사람 (휴대폰/유선전화 사용자)
  • 이 전화는 PSTN(Public Switched Telephone Network, 공중전화망)을 통해 들어옴

2. PSTN → BYOC Premises Edge

  • PSTN은 아날로그 기반 회선이므로 TDM 방식
  • BYOC Premises Edge는 이 전화 회선을 받아 SIP/RTP 기반으로 변환하고 처리함
  • 만약 회선이 TDM이라면 중간에 Gateway 장비가 들어가서 TDM → SIP 변환을 수행한 뒤 Edge로 전달됨

🔧 Edge 장비 역할:

  • SIP 트래픽 처리
  • RTP 미디어 포워딩
  • Genesys Cloud와의 보안 연결 (TLS, HTTPS)

3. Genesys Cloud (중앙 플랫폼)

  • Genesys가 AWS 기반으로 제공하는 클라우드 Contact Center 플랫폼
  • 음성 라우팅, 큐 관리, IVR 처리, 에이전트 연결 등 대부분의 기능이 여기서 처리됨

4. Architect (IVR Design)

  • Genesys Cloud 안에서 전화 흐름(시나리오)을 설계하는 도구
  • 고객이 "1번: 주문, 2번: 배송" 같은 키패드를 누르면, 이 흐름이 Architect에서 정의됨
  • 상담사 연결, 안내 메시지, 조건 분기 등을 처리함

5. API Integration

  • Genesys Cloud는 외부 시스템(CRM, ERP 등)과 데이터를 주고받기 위해 Data Action이라는 기능을 제공
  • Architect 안에서도 Data Action을 통해 외부 API를 호출해서 고객정보를 가져올 수 있음
  • 예: 고객이 전화 → 전화번호로 CRM에서 이름/등급 조회 → 상담사 화면에 자동 표시

6. CRM / External System

  • 외부에 존재하는 고객 정보 저장소 (ex: Salesforce, Oracle ERP, 자체 DB 등)
  • 이 시스템은 Genesys Cloud에서 API를 통해 접근 가능
  • 반드시 Genesys에 직접 붙는 게 아니라, 중간에 API 서버 또는 API Gateway가 있을 수 있음

7. Agent (WebRTC)

  • 상담사는 웹 브라우저 또는 Softphone을 통해 통화에 참여
  • 대부분은 WebRTC 기반으로 Genesys Cloud와 연결됨
  • 이 브라우저 안에서 음성 통화, 고객정보 보기, Wrap-up 등 기능을 수행함

🎧 WebRTC 장점:

  • 별도 전화기 없이 상담 가능
  • 오디오/비디오 품질이 좋음
  • 보안 연결(TLS, SRTP) 가능
