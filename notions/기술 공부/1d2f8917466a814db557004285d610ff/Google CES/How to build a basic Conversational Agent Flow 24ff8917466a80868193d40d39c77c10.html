How to build a basic Conversational Agent Flow?

개요

Conversational Agents는 대화형 에이전트 설계에 대한 간단하고 시각적인 봇 구축 방식을 제공합니다. 봇 디자이너는 이제 전체 봇 구축 프로세스를 훨씬 더 명확하게 파악할 수 있으며, 여러 디자이너가 동일한 에이전트 빌드에 대해 쉽게 협업할 수 있습니다. Conversational Agents는 에이전트가 사용하고 이해할 수 있는 다양한 언어를 지원 하지만, 이 랩에서는 영어만 사용합니다.

이 랩에서는 대화형 에이전트 콘솔을 사용하여 대화형 에이전트를 구축합니다.

필수 조건

대화형 AI의 기본 개념 에 대해 전반적으로 알고 있어야 합니다 . 이 샘플 대본을 읽고 고객 대본이 어떤 형태일지 감을 잡아보세요. 에이전트를 구축하는 첫 단계는 대개 고객 대본 및/또는 기타 상황 데이터를 읽고 사용 사례와 구체적인 비즈니스 요구 사항을 파악하는 것입니다.

작업 2. Intent 생성

인텐트는 최종 사용자가 에이전트와 상호작용하는 이유(예: 주문)입니다. 사용자가 탐색하려는 모든 주제에 대해 인텐트를 만들 수 있습니다.

인텐트는 여러 페이지와 플로우에서 재사용할 수 있습니다. 각 인텐트는 최종 사용자가 일반적으로 묻는 학습 문구로 정의됩니다. 이러한 문구에는 도착 도시나 출발 날짜와 같은 특정 매개변수를 수집하기 위해 주석을 달거나 "레이블"을 지정할 수 있습니다.

대화형 에이전트는 의도에 대한 훈련 문구를 포함할 때 주석을 제안합니다. 또한 최종 사용자와 에이전트의 상호 작용에서 추출하려는 매개변수 값을 수집하기 위해 수동으로 주석을 추가할 수도 있습니다.

권장 사항: 인텐트를 재사용하고 유지 관리를 쉽게 하려면 인텐트에 명확하고 명시적인 이름을 지정하세요.

의도의 형식:category.some_description

서식의 예:

  • 핵심 의도:main.book_a_flight
  • 공통된 의도이지만 핵심은 아니다:supplemental.flight_emissions
  • 재사용 가능한 인텐트: confirmation.yesconfirmation_noredirect.live_agent

작업 3. Flows and Pages

흐름은 주제와 관련 대화 경로를 정의하는 데 사용됩니다. 모든 에이전트는 이라는 하나의 흐름을 갖습니다 Default Start Flow. 간단한 에이전트에는 이 단일 흐름만으로도 충분할 수 있습니다.

더 복잡한 에이전트에는 추가 흐름이 필요할 수 있으며, 다양한 개발팀 구성원이 이러한 흐름을 구축하고 유지 관리할 책임을 맡을 수 있습니다.

모든 흐름은 하나 의 페이지(Page) 로 시작하며 , 이후 하나 이상의 서로 다른 페이지로 구성되어 특정 흐름 내의 대화를 처리합니다. 최종 사용자가 현재 보고 있는 페이지는 활성 페이지(Active Page) 로 간주됩니다 . 각 페이지는 최종 사용자로부터 필요한 정보를 수집하도록 구성할 수 있습니다.

기본 시작 흐름에서 빌드

에이전트가 시작하는 페이지를 기본 시작 흐름(Default Start Flow) 이라고 합니다 . 이 페이지에는 라우팅 로직, 응답( 이행( Fulfillment )이라고 함 ), 인텐트를 일치시킬 수 없거나( 일치하지 않음( no-match )이라고 함 ) 입력이 없을 때 (에이전트가 시간 내에 응답을 받지 못할 때) 수행할 특정 작업이 저장됩니다.

  1. Flows 콘솔 에서 빌드 탭을 클릭합니다.
  1. 시작 페이지를 클릭하여 페이지를 엽니다.
  1. 확장된 옵션에서 경로 옆에 있는 +Start Page 아이콘을 선택합니다 .
  1. 드롭다운에서 main.book_a_flight 인 텐트를 선택한 다음 저장을 클릭합니다 .
  1. 다음으로, 섹션에서 main.book_a_flightRoutes 경로를 클릭합니다 .
  1. 섹션 까지 아래로 스크롤하여 페이지Transition 라디오 옵션을 선택한 다음 드롭다운 메뉴에서 + 새 페이지를 선택합니다.
  1. 페이지 이름을 티켓 정보로 지정 하고 저장을 클릭합니다 .
  1. 창을 종료하고 흐름의 기본 표시로 돌아가서 시작 페이지 에 연결된 새 티켓 정보 페이지를 확인하세요 .

    이제 흐름의 시작 부분에는 인사말이 포함되고, 인텐트가 일치하면 티켓 정보 페이지 로 이동합니다. 이 페이지에서 최종 사용자가 항공편을 예약할 수 있도록 매개변수를 수집합니다. main.book_a_flightTicket information

작업 4. Entities and Parameters

  • 엔티티는 최종 사용자로부터 추출하려는 정보 유형을 정의합니다(예: 도착 도시). 날짜, 시간, 색상, 이메일 주소 등을 일치시키려면 대화형 에이전트에 내장된 " 시스템 엔티티 "를 사용하세요.
  • 시스템 엔터티는 기본 시스템 값에 포함되지 않은 값을 포함하도록 "확장"될 수도 있습니다. 완전히 사용자 지정된 엔터티를 생성해야 하는 경우, 비즈니스에 맞게 사용자 지정되고 시스템 엔터티로 제공되지 않는 데이터와 일치하는 사용자 지정 엔터티 유형을 생성하여 이를 수행할 수 있습니다.
  • 매개변수는 날짜, 시간, 목적지 도시 등 세션 중에 최종 사용자가 제공하는 정보입니다. 각 매개변수는 a name와 entity type.을 가지며, 단어 사이에 밑줄을 사용하여 소문자로 작성됩니다.
