How to build an AI agent with MCP, ADK, and A2A on Google Cloud

  1. Agent는 API를 직접 호출하지 않는다.
  1. Agent는 MCP를 통해 “도구(Tool)”를 사용한다.
  1. Agent는 A2A를 통해 다른 Agent와 협업한다.

Model Context Protocol(MCP)

  • Allow an agent to use tools and access data
  • Read files and execute functions
  • Lets agents use specialized tools

MCP를 한 문장으로 설명하면 : AI 모델이 외부 도구/리소스를 표중 방식으로 사용하기 위한 프로토콜

왜 MCP가 필요한가?

기존 방식의 문제점:

  • Agent가 API 스펙에 강하게 의존함
  • 도구가 늘어날수록 프롬프트가 복잡해짐
  • 보안 / 인증 / 배포를 Agent가 직접 신경 써야 함

MCP의 해결 방식

  • 도구를 별도의 서버(MCP Server)로 분리
  • Agent는 어떤 도구가 있고, 어떻게 호출하는지만 알면 됨.
  • API 변경 → MCP 서버만 수정하면 됨

개념적으로 MCP는 AI 용 API Gateway 또는 Reverse Proxy 역할을 한다.

Agent Development Kit(ADK)

  • The building blocks to create AI agents

Agent2Agent(A2A) Protocol

  • Direct and secure communication between AI agents
  • Meeting point where agents discover and understand each others capabilities
  • Help agents delegate tasks and work together across technology providers

Agent Card란?

Agent의 능력과 역할을 설명하는 ‘명함’

포함 정보는

  • Agent 이름
  • 설명
  • 제공 가능한 Skill
  • 호출 방법

다른 Agent는 Agent Card를 보고

  • 이 Agent가 뭘 잘하는지
  • 어떻게 호출해야 하는지

Overview

This tutorial walks through building a complete agentic system on Google Cloud using three core components:

  1. MCP (Model Context Protocol) – Tool server for agents
  1. ADK (Agent Development Kit) – Framework to build and run agents
  1. A2A (Agent-to-Agent Protocol) – Standard for agent-to-agent communication

By the end, you will have a fully working AI agent that can be called by other agents or programs.

1. Model Context Protocol (MCP)

What is MCP?

  • A standard way for AI models and agents to access tools and resources
  • Acts like a reverse proxy or API gateway for AI tools
  • Decouples agents from direct API integrations

Responsibilities

  • Expose tools
  • Standardize tool calling
  • Secure access to external APIs

2. Google Cloud Environment Setup

Requirements

  • Google Cloud Project
  • Billing enabled
  • Cloud Shell
  • Python 3.10+

Required APIs

  • Cloud Run
  • Cloud Build
  • Artifact Registry
  • Vertex AI

Environment variables are configured using the project ID to ensure all resources are scoped correctly.

3. Building the MCP Server

Purpose

Provide a tool that fetches live currency exchange rates.

Tech Stack

  • Python
  • FastMCP
  • External exchange rate API

Key Concepts

  • @mcp.tool decorator registers a function as an MCP tool
  • Tool logic is encapsulated inside a Python function
  • Server runs locally on port 8080 for testing

Validation

  • Local test script confirms the tool works
  • Successful response verifies MCP server functionality

4. Deploying MCP Server to Cloud Run

Why Cloud Run?

  • Persistent service
  • Auto-scaling
  • Secure by default

Security

  • -no-allow-unauthenticated flag ensures access control
  • Cloud Run Proxy is used for authenticated local testing

Result

  • Secure, production-ready MCP server
  • Logs verifiable via gcloud

5. Building the Agent with ADK

Agent Definition

  • System instructions define agent behavior
  • MCP toolset connects agent to MCP server

Example Rule

  • Agent answers only currency-related questions

Testing

  • ADK Web UI runs on port 8000
  • Queries trigger MCP tool calls automatically

6. Agent-to-Agent Protocol (A2A)

What is A2A?

  • Open protocol for agent collaboration
  • Built on HTTP, SSE, JSON-RPC
  • Framework and vendor agnostic

Key Concepts

  • Agent Card: metadata describing agent capabilities
  • Agents can discover and call each other dynamically

7. Exposing the Agent via A2A

Implementation

  • ADK utility converts agent to A2A-compatible server
  • Agent card is auto-generated
  • Server runs on port 10000

Testing

  • A2A Python client sends requests
  • Successful response confirms agent interoperability

This tutorial demonstrates:
