React

리액트란?

: 사용자 인터페이스를 만들기 위한 Javascript 라이브러리 (SPA 라이브러리)

프레임워크 : 개발을 위한 기본 뼈대.

라이브러리 : 개발에 필요한 도구 또는 그 집합

왜 쓰나?

  1. 트렌드이다.
    1. 굉장히 많이 쓰인다.
    1. 수요가 많다.
    1. 방대한 생태계
  1. 편하다
    1. SPA
    1. 화면을 여러 단위로 쪼개고, 재사용하는 식으로 코드 관리가 용이함
    1. 유용한 라이브러리들이 많음

기존에는 모든 페이지에 진입할 때마다 반복적으로 서버의 페이지에 관련된 리소스를 다운로드 받아야 했다. 이를 방지하기 위해 SPA(Single Page Application) 이 등장함.

서버에서 미리 A, B에 관련된 리소스를 모두 받고 브라우저에서 동적으로 화면을 그린다. 즉 Javascript를 통해 현재 페이지의 html을 생성하는 것임.

리액트 특징

  • 컴포넌트 기반 설계. 컴포넌트(스스로 상태를 관리하는 캡슐화된 코드 조각)
    • 의미단위로 컴포넌트 구성
    • 재사용성과 유지보수성 증가
    • 부모, 자식 관계를 가짐
  • 가상돔(Virtual DOM)
    • 가상돔을 하나 두고, 변경되어야 할 돔 요소를 미리 가상돔에서 변경해두고, 이것을 돔에서 동기화시키는 방식으로 함.
  • CSR(Client Side Rendering) : Javascript를 통해 서버가 아닌 클라이언트 즉 브라우저에서 화면을 렌더링한다.
  • 풍부한 생태계
    • 단순 화면 개발 뿐만 아니라 React Native와 같은 앱개발을 위한 라이브러리도 존재함

JSX란

: 자바스크립트를 확장한 문법으로 자바스크립트 XML 이라는 문법임.

JSX = HTML + Javascript

const element = <h1>Hello, world!</h1>

JSX는 리액트를 통해 최종적으로 HTML로 화면에 렌더링됨

JSX 특징

  • JSX에서 사용되는 태그의 속성 이름이 HTML과 조금 다름
    • class → className
    • for → htmlFor
    • onclick → onClick
  • 태그를 명시적으로 닫아줘야 함
    const element = <h1>Hello, World</h1><button>btn</button> // 이렇게 안됨

const element = <div><h1>Hello, World</h1><button>btn</button></div> // 이건 됨
  • 하나의 태그로 감싸져 있어야 함

JSX에서 조건문

반복문 익명 함수를 넣어서 구현함.

			<li>
        {arr.map((item, index) => {
          return <h4 key={index}>{item}</h4>
        })}
      </li>

컴포넌트

import 할때 무조건 대문자로 시작해야한다. import Hello from ‘./Hello.js’

  • 컴포넌트는 본인의 로직과 내부상태(지역변수)를 고유하게 들고 있을 수 있다.
  • 컴포넌트 이름은 항상 PascalCase로 지어야 한다. ex) MyComponent, ArticlePage 등등
  • 컴포넌트는 의미단위로 쪼개준다. 명확한 범위와 의미를 지닐 수 있도록
  • 최상위 컴포넌트 이름은 일반적으로 App이다.
  • index.js는 모든 React 코드의 엔트리 포인트라고 할 수 있고, App.js는 모든 컴포넌트의 루트 컴포넌트라고 할 수 있다.

Props란?

: 부모 컴포넌트에서 자식 컴포넌트로 내려주는 데이터

상위 컴포넌트에서 하위 컴포넌트로 어떤 값을 전달해줘야 할 때 props를 사용하여 전달한다. 

function App() {
	return (
		<div>
			<MyComponent value={'test'}/>
		</div>	
		);
	}

function MyComponent(props) {
	return <div>{props.value}</div>
}

오브젝트 형태로 전달해줄 때의 속성 이름과 동일한 키로 접근하여 사용할 수 있다. value라는 키로 넘겨주니까 MyComponent에서도 value라는 키로 넘겨 받고 있다.

props 활용 팁.

  1. 구조 분해 할당 구문을 잘 활용하자.
  1. 특정 props에 기본 값을 줄 수 있다. (defaultProps)
  1. props는 읽기 전용이다.

state란?

: 컴포넌트 내부에서 사용되는 변수.

state 값이 변하면 컴포넌트가 리렌더링 된다. 렌더링 사이클에서 값이 보존된다.

useState() 함수가 반환하는 타입은 배열이고, 배열의 첫번째 값은 State 즉 value가 되는 것이고, 두번째 값은 앞에있는 state를 변화시키는 setter 함수이다.

import logo from '../logo.svg';
import Heading from './Heading';
import { useState } from 'react';

export default function App() {
  const [value, setValue] = useState(0);  // useState의 인자는 value의 초기값이 들어간다
  
  return (
    <div>
      <h1>value: {value}</h1>
      <button onClick={()=>{
        setValue(value+1);
        console.log('current value: ', value);
      }}>Increase Value</button>
      
      <button onClick={()=>{
        setValue(0);
      }}>Reset Value</button>
    </div>
  );
}

클래스형 컴포넌트

== 클래스 문법으로 구현한 컴포넌트.

  • useState와 같은 Hooks는 React 버전 16.8 부터 등장
  • 즉, 그전에는 함수형 컴포넌트에서 state를 사용할 수 없었고, 클래스형 컴포넌트만 state를 가질 수 있었다.
  • 클래스의 멤버 변수로 state 정의
  • render라는 멤버 함수에서 반환한 값 JSX 가 화면에 그려짐

클래스 컴포넌트는 클래스의 멤버 변수로 state를 정의하면 화면을 그릴 때는 render라는 멤버 함수를 넣어서 해당 함수가 반환되는 값을 화면에 그린다.

클래스가 컴포넌트가 되기 위해서는 리액트에 있는 컴포넌트라는 클래스를 상속받아야 한다.

createRef는 리액트에서 컴포넌트가 마운트될 때, current 프로퍼티에 DOM 요소를 할당하고, 컴포넌트가 언마운트될 때 current 프로퍼티에 다시 null 을 할당한다.
