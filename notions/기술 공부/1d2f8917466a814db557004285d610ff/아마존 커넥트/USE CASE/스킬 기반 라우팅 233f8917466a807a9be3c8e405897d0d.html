스킬 기반 라우팅

Amazon Connect Skill-Based Routing Flow 구성 정리

<전체 목적>

고객의 Saving 역량 요구 수준에 따라 적절한 상담원(Routing Criteria의 스킬 수준 기반)에게 연결하는 Skill-Based Routing 시나리오를 구현.

주요 블록별 역할

1. Set Logging Behavior (UpdateFlowLoggingBehavior)

  • 역할: Connect Contact Flow 실행 로그를 CloudWatch에 남기도록 설정
  • 중요 이유: 디버깅 및 오류 추적에 필수

2. Check Hours of Operation

  • 역할: 현재 시간이 근무 시간(HoursOfOperation) 내에 있는지 확인
  • 조건 분기:
    • True: 다음 인사 메시지로 이동
    • False: “not working hour” 메시지 후 연결 종료

3. Set Contact Attributes (UpdateContactAttributes)

  • 역할: 현재 컨택에 고객이 원하는 Saving 수준(예: 2, 3, 4 등)을 설정
  • 형식:
    "Attributes": {
  "Saving": "2"
}

4. Set Routing Criteria (UpdateContactRoutingCriteria)

  • 역할: 현재 컨택에 대해 어떤 상담원과 연결할지 라우팅 기준 설정
  • 조건 예시:
    • Attribute: Saving
    • ProficiencyLevel: 5
    • ComparisonOperator: NumberGreaterOrEqualTo
    • → Saving >= 5인 상담원만 매칭 가능

5. Invoke Lambda Function

람다 함수는 Amazon Connect 콘솔 홈에서 먼저 등록을 해줘야 목록에 뜬다.

  • 함수명: sb-connectTestFunction
  • 역할: 고객의 Saving 값이 조건에 적합한지 Lambda에서 로직 판단
  • Lambda 반환값 예시:
    {
  "statusCode": 200,
  "savingCheckResult": "allow" | "deny"
}

6. Compare (조건 분기)

  • 역할: Lambda 함수의 결과(savingCheckResult)를 평가
  • 조건:
    • allow → Transfer to Queue
    • deny → 메시지 후 연결 종료

7. Transfer to Queue

  • 큐 이름: sb-queue-test
  • 역할: 스킬 기준을 충족하는 상담원에게 연결 시도

💡 핵심 개념 요약

개념설명
Contact Attribute고객에게 지정되는 속성 값 (ex. Saving 수준)
Routing Criteria상담원과 연결할 조건 (스킬 이름, 수준 등)
ProficiencyLevel상담원의 역량 수준 (1~10)
Lambda Function외부 로직 처리 (조건 분기, 판단, API 연동 등)
Compare 블록Lambda 결과값을 기준으로 분기 처리
Queue연결 가능한 상담원 그룹

✅ 고객 → 상담원 연결 흐름 요약

  1. 고객 인입
  1. 근무 시간 확인
  1. 고객이 원하는 Saving 수준 설정 → Set contact attributes —> 람다 함수를 호출하기 위해 필요.
  1. 상담원 연결 기준 (Routing Criteria) 설정
  1. Lambda에서 Saving 값 판단
  1. Compare 블록으로 결과 분기
    • allow: 상담원에게 연결
    • deny: 안내 메시지 후 종료

🧪 테스트 팁

  • Lambda 함수 내 로그: CloudWatch > Log Group > /aws/lambda/{function name}
  • Contact Attributes 확인: Flow > Test 모드 > Contact Summary
  • 상담원 스킬 설정: Amazon Connect Admin > Routing > Users > Agent > Routing Profiles
