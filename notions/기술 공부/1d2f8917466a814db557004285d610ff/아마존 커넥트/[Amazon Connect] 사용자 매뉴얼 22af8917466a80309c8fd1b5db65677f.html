[Amazon Connect] 사용자 매뉴얼

Amazon Connect 주요 구성요소

Amazon Connect는 AWS에서 제공하는 클라우드 기반 컨택센터 플랫폼으로, 음성/채팅 기반 고객 응대를 자동화하거나 상담사에게 효율적으로 라우팅할 수 있는 다양한 기능을 제공합니다. 아래는 Amazon Connect의 핵심 기능들을 실무적으로 이해하기 위한 상세 설명입니다.

1. Contact Flow: 고객 응대 흐름을 설계하는 중심 기능

개요

Contact Flow는 고객이 전화를 걸었을 때부터 종료할 때까지 어떤 순서로 응대를 진행할지 시나리오를 정의하는 핵심 기능입니다. Amazon Connect의 모든 응대 프로세스는 이 Flow 안에서 설계됩니다.

특징

  • 비개발자도 사용 가능한 시각적 편집기(UI) 제공
  • “블록 단위”로 구성되며, 각 블록은 조건, 안내, 분기, 전송 등의 역할을 수행
  • 실제 전화 흐름, 시간대 분기, 고객 입력 처리, 상담사 연결, 외부 시스템 연동 등을 포함

자주 사용하는 주요 블록

블록 명설명
Start모든 Flow의 시작점
Play Prompt고객에게 음성 안내를 재생 (TTS 또는 오디오 파일)
Get Customer Input고객의 숫자 입력을 받아 조건 분기 (IVR 메뉴)
Check Hours of Operation현재 시간이 운영 시간인지 판단하여 흐름 분기
Transfer to Queue특정 큐(Queue)에 연결하여 상담사에게 라우팅
Disconnect / Error Handling흐름 종료, 실패 시 대체 경로로 전환
Set Voice음성 언어 및 목소리 설정
Invoke AWS Lambda외부 API 또는 백엔드 시스템 연동을 위한 Lambda 함수 호출

활용 예시

“안녕하세요, 고객센터입니다. 1번은 주문 문의, 2번은 환불 요청입니다.”

→ 고객이 1을 누르면 주문 큐, 2를 누르면 환불 큐로 전송

→ 각각에 해당하는 라우팅 프로필을 가진 상담사에게 자동 연결

2. 🔊 Play Prompt & Voice Prompt: 음성 안내 구성

Play Prompt란?

고객에게 음성 안내를 들려주는 블록입니다. 안내 멘트는 다음과 같은 방법으로 설정할 수 있습니다.

① TTS (Text to Speech)

  • 텍스트를 입력하면 Amazon Polly가 자동으로 음성으로 변환
  • 한국어는 Seoyeon 목소리가 자연스러워 많이 사용
  • 빠른 시나리오 테스트, 반복 편집이 유리

② 오디오 파일

  • .wav 파일 형식으로 녹음된 음성을 업로드하여 사용
  • 고유 브랜드 보이스 또는 감정 전달이 중요한 경우 선호
  • S3에 업로드한 오디오 파일을 Contact Flow에서 지정 가능

Play Prompt 활용 예

  • "안녕하세요. XX 고객센터입니다. 상담 연결까지 잠시만 기다려 주세요."
  • "운영 시간은 평일 오전 9시부터 오후 6시까지입니다."

3. 🎧 Queues (대기열): 고객을 대기시키고 라우팅하는 중간 단계

정의

큐는 고객이 상담사와 연결되기 전까지 머무는 가상의 공간입니다. Contact Flow에서 Transfer to Queue 블록을 사용해 고객을 큐에 배정할 수 있습니다.

주요 기능

  • 큐별로 안내 메시지, 대기음 설정 가능
  • 큐별 통계 확인: 대기 중 고객 수, 평균 대기 시간 등
  • 큐별 우선순위 설정: 더 중요한 큐는 빠르게 연결되도록 구성 가능
  • 큐는 상담사 연결의 중간 지점이며, Routing Profile과 연계되어 실제 상담사에게 전달됩니다.

예시

  • 일반 문의 큐: 응답 시간은 느릴 수 있으나 많은 상담사 보유
  • VIP 큐: 소수의 전문 상담사 배정, 높은 우선순위로 처리

4. 🧭 Routing Profile (라우팅 프로필): 어떤 상담사가 어떤 큐를 받을지를 정하는 설정

정의

Routing Profile은 상담사에게 어떤 큐의 고객을 연결할 것인지 정의하는 역할을 합니다. 상담사(User)에게 이 프로필을 할당함으로써, 자동 라우팅이 가능해집니다.

구성 요소

항목설명
큐 목록이 상담사가 받을 수 있는 큐들
채널음성, 채팅 등 처리 가능한 채널
동시 처리 수상담사가 동시에 처리 가능한 콜 수
우선순위상담사가 어떤 큐의 콜을 먼저 받을지 결정하는 기준

예시

  • 일반 상담사는 ‘주문 큐’, ‘배송 큐’만 연결됨
  • 기술 상담사는 ‘기술지원 큐’만 받도록 설정

5. 👥 Users (상담사 / 관리자): 실제 Amazon Connect를 사용하는 사람들

정의

Amazon Connect 시스템에 로그인하여 실제 전화를 받고 상담하는 인원이 User입니다.

사용자 유형

  • Agent (상담사): 고객 전화를 수신, 응대
  • Admin (관리자): 설정 변경, 사용자 관리, 리포트 확인 가능

사용자 구성 시 설정 필요 사항

  • 라우팅 프로필: 어떤 큐를 받을 것인지
  • 보안 프로필: 권한 설정 (예: 상담사, 관리자 등)
  • 팀 구성: 관리 목적의 그룹화 (보고서 필터링 등)

상담사 인터페이스

  • 브라우저 기반 Agent UI 사용
  • 상태 설정 가능 (대기, 통화 중, 자리 비움 등)
  • 실시간 콜 수신 및 통화 관리

6. 🕒 Hours of Operation (운영 시간)

정의

각 Contact Flow 내에서 운영 시간 조건에 따라 흐름을 제어하는 블록입니다.

주요 기능

  • 요일 및 시간 단위로 설정 가능 (예: 평일 09:00~18:00)
  • Contact Flow에서 Check Hours of Operation 블록과 함께 사용
  • 운영 시간 외에는 다른 안내멘트 또는 종료로 분기 가능

활용 예시

  • 운영 시간 중: “상담사와 연결 중입니다.”
  • 운영 시간 외: “현재 운영시간이 아닙니다. 평일 오전 9시부터 오후 6시까지 운영합니다.”

7. ☎️ Phone Number: 실제 고객이 전화를 거는 번호

정의

고객이 Amazon Connect에 접속하기 위한 입구입니다. 전화번호마다 다른 Contact Flow를 지정할 수 있습니다.

설정 항목

  • 국가별 DID (직접 연결) 번호 또는 Toll-free 번호 사용 가능
  • 하나의 번호에 하나의 Contact Flow 연결
  • 예: 080-1234-5678 → Welcome Flow 실행

8. 📊 모니터링 및 리포트 기능

실시간 모니터링

  • 상담사 상태 (대기 중, 통화 중 등)
  • 큐별 대기 인원
  • 평균 응답 시간, 연결률 확인

리포트

  • 큐별 / 팀별 통화량 분석
  • 응답률, 이탈률, 상담사 성과 측정
  • CSV 다운로드 및 Amazon QuickSight 등과 연계 가능

AWS 서비스와의 통합

✅ 실무 흐름 예시 시나리오

1. 고객이 전화번호 080-1234-5678로 전화
2. 해당 번호에 연결된 Contact Flow 실행
3. "운영 시간 확인" → 현재 시간 기준 분기
   - 운영 시간 외: 운영 종료 안내 후 Disconnect
   - 운영 시간 내: "안녕하세요 고객센터입니다" Play Prompt
4. 고객이 '1' 입력 → 주문 큐로 전송
5. 해당 큐를 포함하는 Routing Profile 가진 상담사에게 자동 연결
6. 상담사가 Agent UI에서 통화 수신

블록 라이브러리 > 상호 작용(Interactions) 섹션 주요 블록 설명

1️⃣ 고객 인증 (Get customer input for Amazon Connect Voice ID)

고객의 음성이나 입력을 통해 Amazon Connect Voice ID를 기반으로 신원 인증을 수행합니다.

🔹 용도:

  • 생체 기반 인증 (예: 음성 패턴)
  • 사전 등록된 Voice ID가 필요

🔹 예시:

고객이 “고객센터입니다”라고 말하면 → Voice ID로 본인 확인

→ 인증 성공 시 $.Attributes.Authenticated = true 저장됨

이 블록은 Amazon Connect Voice ID 서비스를 활성화한 경우에만 사용 가능

2️⃣ 고객 입력 가져오기 (Get customer input)

고객으로부터 숫자나 음성 입력을 받는 블록입니다.

🔹 용도:

  • 키패드 입력 (DTMF) 또는 음성 인식으로 고객 응답 받기

🔹 예시:

“문의하실 항목의 번호를 눌러주세요.

1번: 요금 문의, 2번: 해지 문의”

  • 고객이 2번을 누르면 →

    $.CustomerInput = "2"로 저장됨

🔹 주요 설정:

  • 입력 방식 선택: DTMF (터치패드), 음성, 둘 다
  • 입력 값 제한 (예: 1~3 자리)

3️⃣ 고객 입력 저장 (Store customer input)

입력된 값을 속성으로 저장하는 블록입니다.

🔹 용도:

  • 고객 입력 가져오기 블록에서 받은 입력을 명시적으로 contact attribute로 저장

🔹 예시:

  • 고객이 1을 눌렀다면:
    $.CustomerInput = "1"
  • 이걸 CallType 이라는 속성으로 저장:
    Set attribute: $.Attributes.CallType = $.CustomerInput

🔹 사용 포인트:

  • 다른 블록(예: 조건 분기, Lambda)에 입력값 전달하려면 반드시 이 블록을 거쳐야 함

4️⃣ 메시지 보내기 (Send message)

채팅(Contact Channel = CHAT)에서 고객에게 텍스트 메시지를 보내는 블록입니다.

🔹 용도:

  • 웹 채팅, SMS 등 비음성 채널에서 자동 응답 발송

🔹 예시:

고객이 채팅을 시작하면

→ “안녕하세요, 무엇을 도와드릴까요?” 자동 전송

⚠️ 주의:

  • 음성 기반 Contact Flow에서는 이 블록은 작동하지 않음

5️⃣ 재생 프롬프트 (Play prompt)

고객에게 음성 메시지를 재생합니다. Contact Flow에서 가장 자주 사용됨

🔹 용도:

  • 안내 멘트, 인증 요청, 선택지 안내 등 모든 음성 출력

🔹 예시:

  • "상담원을 연결해 드리겠습니다"
  • "1번은 요금 문의, 2번은 해지 문의입니다"

🔹 구성 방식:

  • Text-to-Speech (TTS, 예: Polly)
  • 또는 미리 녹음한 오디오 파일 (S3에 저장된 .wav 등)

6️⃣ 지속적인 연락처 연결 생성 (Create persistent contact association)

고객과의 연결을 지속시키기 위한 **연관(Contact association)**을 설정

🔹 용도:

  • 특정 고객의 세션/히스토리 유지를 위해 사용
  • *고객 ID(Customer profile)**와 현재 Contact를 연결

🔹 예시:

고객이 로그인 시 ID(예: customer-001)를 설정하면

→ 해당 ID로 향후 대화 내역을 추적 가능

⚠️ 일반적인 Contact Flow에서는 자주 사용되지 않음

→ Customer Profiles 기능을 함께 활성화한 경우 사용

7️⃣ 태스크 생성 (Create task)

Contact Flow 중 특정 조건일 때 작업(Task)을 생성하여 후속 조치 처리

🔹 용도:

  • 상담사가 나중에 처리해야 할 업무를 생성
  • 예: 전화 통화 중 처리 불가한 요청을 메모하여 Task로 등록

🔹 예시:

고객이 "기술팀에게 전달해주세요" 라고 했을 때 →

→ Task 생성 → 상담사 인박스에 할당

🔹 구성 요소:

  • Task 이름
  • 할당 대상 (Queue or agent)
  • 속성 (ex: TaskType, Priority 등)
