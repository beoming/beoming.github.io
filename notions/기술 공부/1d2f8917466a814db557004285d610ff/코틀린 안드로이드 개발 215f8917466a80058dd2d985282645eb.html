변수선언

val name = "권신범" // 변경 불가함 
var age = 28 // 변경 가능함

var name : String = "길동"

코틀린은 별도로 타입을 지정해주지 않아도 된다. 타입 추론 기능이 있기 때문이다.

타입을 지정해주고 싶을 경우, 변수이름 뒤에 콜론 : 을 붙여주고 타입을 지정해줄 수 있다. 코틀린에서는 모두 레퍼런스 타입(Reference Type) 즉, 대문자로 시작하는 타입 형태만 사용하고 있다.

상수는 val 이라는 이름을 사용하고 자바의 final과 동일하다고 생각하면 된다. val은 런타임 상수이므로 변수의 초기화 값이 런타임에 결정된다.

컴파일 타임 상수 ⇒ 프로그램이 실행되기 전에 컴파일 시점에 값이 결정되는 상수

const val num = 20;  // 컴파일 타임 상수

fun main() {
	var i : Int = 10
	var name : String = "신범"
	var point : Double = 3.3
}

const 변수는 값을 변경할 수 없으며, 컴파일 타임 변수이므로 선언과 동시에 초기화되어야 한다.

물음표 느낌표

? → Null 일 수 있음. 변수에 null 값을 허용하겠다는 뜻

var name: String? = "영희"
name = null // 가능

! !→ 절대 Null이 아님. 개발자가 단정하고 강제로 실행하는 연산자

val name: String? = null
val length = name!!.length // NullPointException 발생

형변환

코틀린은 자동 형변환을 지원하지 않으며, (타입)과 같은 캐스팅 연산을 지원하지 않는다. 

fun main() {
	var i = 10
	var l = 20L
	var name = "10"
	i = name.toInt()
	
	l = i.toLong()
	i = l.toInt()
}

toByte(), toShort(), toInt(), toLong(), toFloat(), toDouble(), toChar() 등의 함수를 사용하여 기본 타입간의 형변환을 수행할 수 있다.

val obj: Any = "Hello"
if(obj is String) {
	val str: String = obj as String // 스마트캐스트 또는 명시적 형변환
}
  • as 연산자를 사용하여 명시적으로 형변환을 수행한다.
  • is 연산자를 사용하여 타입 체크를 먼저 수행한 후 as 연산자를 사용하는 것이 안전하다
  • 스마트 캐스트란 코틀린 컴파일러가 is 연산자 이후 특정 조건에서 자동으로 형변환을 수행하는 것을 말한다.
  • Any 타입은 코틀린의 최상위 타입으로 자바의 Object 타입과 유사하다. 모든 코틀린 클래스는 암시적으로 Any를 상속한다.

[문자열의 동등 비교]

Kotlin에서는 Java와 달리 == 연산자를 이용하여 문자열의 ‘값’이 같은지 비교할 수 있다. 또한 문자열의 참조 주소를 비교하고 싶다면 === 연산자를 사용하여 비교할 수 있다.

[문자열의 사전순 비교]

두 문자열 중 어느것이 사전순으로 앞서는지만 비교하고 싶을 경우에 간단히 부등호만 이용하면 된다. println(”abcd는 abce보다 사전순으로 앞섭니다 : “ + (str1 < str2))

[문자열 템플릿]

문자열들을 합칠 때 + 연산자를 사용하는 대신 문자열 내부에 달러표시($)와 중괄호를 사용한다.

val a = 2
val b = 3
println("$a + $b = ${a + b}") // 2 + 3 = 5

// 중괄호 안에 있는 수식은 계산에 반영하는 수식이다.

[특정 인덱스의 문자 가져오기]

Kotlin에서는 charAt() 메서드를 아예 지원하지 않고, 배열의 요소를 가져오듯 대괄호안에 인덱스 값만 넣어주면 된다.

String str = "Hello World"
println(str[1]) // e

[문자열 잘라서 추출]

Kotlin에서는 문자열을 잘라서 추출하는 방법으로는 substring() 메서드와 slice() 메서드가 있다. substring() 메서드는 Java와 내부적으로 동일한 메서드이고, slice()는 Collection 객체에서 일부를 추출하는 메서드이지만 문자열에서는 substring()과 비슷하게 동작한다.

val str = "Hello World"
println(str.substring(3)) // lo World
println(str.substring(3, 7)) // lo W
println(str.substring(3 until 7)) // lo W

CallBack 함수

어떤 함수의 파라미터로 들어가는 함수를 콜백 함수라 한다.

특정 함수의 파라미터로 넘겨받아, 특정 작업이 끝난 뒤에 콜백 함수를 호출하여 특정 작업이 끝났을 떄의 이벤트에 대한 처리를 할 수 있다.

대표적인 예시로, 안드로이드의 클릭 리스너가 있다. 사용자의 클릭을 감지하면 onClickListener를 호출하여 사용자가 클릭에 대한 이벤트를 처리할 수 있도록 하는 것.

즉, 특정 작업이 끝났다는 것을 알려주기 위해 사용된다.

  • xml에서 정의한 View를 Kotlin 코드와 연결하는 기본 방식

리스트 뷰(List View)

MainActivity → [”A”, “B”, “C”] → Adapter → listview_item.xml → activity_main.xml (listview)

ListView는 이름에서도 알 수 있듯이 리스트 즉, 목록을 구현하는데 사용된다. ListView는 안드로이드에 임베디드 되어 있는 코드로 동작하며 API level 1부터 존재했다.

getView()의 접근 방법으로 동작하게 되면 ListView의 재사용성이 떨어진다. 예를들어 아이템이 20개가 있고 이를 스크롤 한다고 가정하면 스크롤 시에도 getView()함수는 계속해서 호출된다.

Holder holder = new Holder()
View rootView = inflater.inflate(R.layout_item_list, null)

위의 코드는 별도의 null 처리가 없으므로 스크롤을 할 때마다 inflate를 통해서 View의 create가 발생하고 findViewByid도 함께 호출된다. ⇒ 비효율적.

inflate와 findViewById를 리스트뷰에서 연속적으로 발생시키면 메모리와 성능에 악영향을 미칠 수 있다.

Intent, DataBinding, Adapter 개념

Intent, DataBinding, Adapter 이 세 가지 개념은 각각의 역할이 다르고 서로 다른 레이어에서 작동하지만, 앱을 구성하는 데 있어 매우 중요한 역할을 합니다.

✅ 개념 요약 도식 (전체 흐름)

[Activity A] ── (Intent) ──▶ [Activity B]
   │                             │
   ▼                             ▼
[DataBinding]             [Adapter + ListView/RecyclerView]
   │                             │
[XML View ↔ Data]        [View ↔ Data List]

① Intent

🔹 개념

Intent는 컴포넌트 간의 통신을 위한 객체입니다. 예를 들어, 한 화면(Activity)에서 다른 화면으로 이동할 때 사용합니다.

🔹 역할

  • 액티비티 간 전환 (Activity → Activity)
  • 서비스 시작
  • 브로드캐스트 전송
  • 데이터 전달 (putExtra()로 값 전달)

🔹 예시 코드

val intent = Intent(this, DetailActivity::class.java)
intent.putExtra("username", "shinbeom")
startActivity(intent)

이렇게 하면 DetailActivity에서 intent.getStringExtra("username")으로 값 받아올 수 있음.

② DataBinding

🔹 개념

DataBinding은 XML 뷰와 Kotlin(Java) 데이터 간의 연결을 자동화해주는 기술입니다. 뷰를 코드에서 직접 findViewById()로 찾지 않아도 됩니다.

🔹 역할

  • 레이아웃 XML과 데이터 객체를 직접 연결
  • 코드의 간결화
  • LiveData/Observable과 연계해 MVVM 패턴 구현 가능

🔹 사용 준비

  • build.gradle에 dataBinding 활성화
android {
    ...
    buildFeatures {
        dataBinding = true
    }
}
  • XML에 <layout> 태그로 감싸기
<layout xmlns:android="http://schemas.android.com/apk/res/android">
    <data>
        <variable
            name="user"
            type="com.example.User" />
    </data>

    <TextView
        android:text="@{user.name}" />
</layout>
  • 코드에서 바인딩 객체 생성
val binding = DataBindingUtil.setContentView<ActivityMainBinding>(this, R.layout.activity_main)
binding.user = User("홍길동")

③ Adapter

🔹 개념

Adapter는 리스트 뷰(ListView, RecyclerView 등)와 실제 데이터(List 등)를 연결하는 중간 다리입니다.

🔹 역할

  • 리스트 형태의 뷰가 데이터를 반복해서 표시할 수 있도록 함
  • 각 아이템마다 어떻게 보여줄지 정의

🔹 주요 메서드 (ListView 기준)

메서드설명
getCount()아이템 개수
getItem()위치에 해당하는 데이터
getView()해당 위치의 View 생성 및 재사용

🔹 예시

class MyAdapter(val items: List<User>) : BaseAdapter() {
    override fun getCount(): Int = items.size

    override fun getItem(position: Int): Any = items[position]

    override fun getItemId(position: Int): Long = position.toLong()

    override fun getView(position: Int, convertView: View?, parent: ViewGroup): View {
        val view = convertView ?: LayoutInflater.from(parent.context).inflate(R.layout.item_user, parent, false)
        val nameView = view.findViewById<TextView>(R.id.name)
        nameView.text = items[position].name
        return view
    }
}

버전

Notion의 API 버전은 버전이 론칭된 날짜에 따라 이름이 지어집니다. 예를 들어, 최신 버전은 2022-06-28입니다.

버전은 Notion-Version 헤더를 포함하여 설정합니다. Required에서 Notion-Version 헤더를 설정하세요.

curl https://api.notion.com/v1/users/01da9b00-e400-4959-91ce-af55307647e5 \
  -H "Authorization: Bearer secret_t1CdN9S8yicG5eWLUOfhcWaOscVnFXns"
  -H "Notion-Version: 2022-06-28"

객체

사용자

사용자 객체가 API에 나타나는 위치

사용자 객체는 다음을 포함해 API에서 반환되는 거의 모든 객체에 표시됩니다.

  • 생성자, 최종 편집자 속성의 블록 객체
  • 생성자, 최종 편집자 , 사람 속성의 페이지 객체
  • 생성자, 최종 편집자 속성의 데이터베이스 객체
  • 사용자가 멘션 시 서식 있는 텍스트 객체
  • 사람 속성에서 속성 객체

사용자 객에는 아래 설명된 대로 object와 id 키가 항상 포함됩니다. 사용자가 서식 있는 텍스트나 페이지 속성 컨텍스트에서 렌더링되고, 봇이 해당 속성에 액세스할 수 있는 올바른 기능이 있는 경우 나머지 속성이 표시될 수 있습니다. 기능에 대한 자세한 내용은 기능 가이드와 인증 가이드를 참고하세요.
